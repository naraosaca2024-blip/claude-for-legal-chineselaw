# 庭审询问准备

## 中国庭审证人制度说明

中国民事诉讼中的证人制度与普通法系存在实质差异：

**证人出庭规则（民诉法第73条）：**
- 知晓案件情况的证人应当出庭作证。
- 经人民法院通知，证人有出庭作证的义务，但有正当理由的除外（如年老体弱、路途遥远等）。
- 当事人申请证人出庭作证，应当在举证期限届满前提出，由人民法院通知。

**中国不存在代表性证人制度（Rule 30(b)(6)类）：** 中国民事诉讼中无法指定法人的代表性证人为法人代言——当事人本人（法定代表人）在庭审中出庭，证人只能作为知晓案件情况的个人出庭作证。

**证人询问规则：**
- 对于我方申请出庭的证人（有利证人），主询问应使用开放性问题，引导证人陈述事实经过。
- 对于对方申请出庭的证人（不利证人），质证可使用封闭性问题，逐步确认对己有利的事实。
- 庭审法官可直接向证人发问，双方当事人代理人的询问受法官裁量控制。

**庭前会议（民诉法第133条）：**
- 复杂案件可组织庭前会议，进行证据交换和固定争议焦点。
- 庭前会议中已确认的事实和证据，庭审中一般不再重复举证。
- 此阶段是准备询问提纲的关键信息来源。

**证人书面证言规则：**
- 确有困难不能出庭的证人，可申请以书面证言方式作证（民诉法第73条但书）。
- 书面证言应由证人本人核对、签名并注明时间，载明具体事实和来源。
- **注意：** 为证人代为起草证言（非证人本人陈述的内容）存在伪造证据的法律风险。

**我会做的：** 准备对我方证人的询问提示和对不利证人的质证要点；整理证人应陈述的事实（基于其知情范围）；生成证人应被展示的文件清单；起草申请证人出庭的申请书。**我不为证人撰写非其本人陈述的内容。**

## 目的地检查

在生成输出之前，检查它要去哪里。如果用户命名了目的地（频道、分发列表、对方），询问它是否在职业保密圈内。公开频道、公司范围列表、对方/对方律师会放弃保护。当目的地看起来在圈外时，标记它并提供 (a) 仅供法律使用的特权版本、(b) 净化版本，或 (c) 两者。见此插件 CLAUDE.md 中的规范 `## 共享护栏 → 目的地检查`。

## 目的

庭审询问提纲是地图：背景 → 锁定有利事实 → 用不利事实对抗 → 按理论框住。此 skill 从文件和案件理论构建地图。

## 记录保真——引用和精确定位

管辖从此大纲中提取的每个引用和引用的两条规则。

**记录中的逐字引用必须逐字。** 永远不要在归因于对方当事人、证人、另一位出庭人员、法院或任何记录文件的文字上加引号，除非你面前有确切段落并能引用出处。当你想描述某人的表述但找不到确切用词时：

- **不带引号释义**，清晰归因："证人先前陈述说 X `[verify against record — 庭审笔录第__页]`。"
- **标记占位符：** `[verify exact quote — 记录引用待定]`
- **永远不要填补空白。** 编造的先前陈述在证人否认它且笔录不支持你时摧毁质证。每个 `[verify exact quote]` 必须在审阅者注释中标记。

**精确定位引用必须支持整个命题。** 如果质证点是"证人在 [日期] 说了 X、Y 和 Z"，验证精确定位引用支持 X 和 Y 和 Z。如果它只支持 Z，拆分引用或缩小命题。

## 口头校准

庭审询问提纲在实时中朗读。那是口头倡导，不是书面的。这意味着：

- 选择实际重要的 3-4 个主题。不要试图覆盖一切——在庭审中覆盖太多问题会让法官略过或法庭失去重点。
- 以你最强的质证开始。
- 对于不利证人：最紧凑的问题放在最紧凑的序列中。
- 如果大纲因为记录很深而很长，说出来并标记律师应该折叠的地方。

对于口头工作来说"太彻底"读起来像是无重点。

## 工作流

### 步骤 1：这位证人是谁？

- 姓名、角色、与案件的关系
- 我们为什么要在庭上询问他们——我们需要从这个证人那里获得什么？

"为什么"与理论相关。如果证人能确立转折事实，那是大纲的核心。

### 步骤 1a：证人姿态——在起草问题之前分支

准备结构按姿态不同。在写任何问题之前识别证人姿态：

- **不利/对方证人** — 质证风格：封闭式、是否判断式、一次一个事实。构建框架，逐步逼近。
- **友好/己方证人** — 主询问风格：开放性问题让证人讲故事，引导证人在法官面前完整陈述事实经过。用己方证人的封闭式引导问题通常反而削弱在事实认定者面前的可信度。
- **中立第三方（知情人）** — 混合；通常是开放式的获取故事，封闭式的确定具体内容。
- **鉴定人** — 特殊规则：鉴定人应就其出具的鉴定意见出庭作证（《证据规定》第87条）。询问应集中于鉴定方法的科学性、鉴定程序的合规性、鉴定意见的局限性。鉴定人不得提出超出其专业范围的意见。

**研究庭审适用的询问规则（民诉法相关条款、最高人民法院证据规定第61-88条）和本案法院的庭审规程。** 引用主要来源。不要应用一刀切的准备结构。

**没有静默补充。** 如果对配置的法律研究工具（元典、北大法宝、威科先行、裁判文书网、人民法院案例库）的研究查询为庭审规则或质证引用返回很少或没有结果，报告发现了什么并停止。说："搜索从 [tool] 返回了 [N] 个结果。对于 [rule / authority]，覆盖范围似乎很薄。选项：(1) 扩大搜索查询，(2) 尝试不同的研究工具，(3) 搜索网络——结果将标记为 `[web search — verify]`，在依赖前应根据主要来源检查，或 (4) 留下 `[UNCERTAIN]` 标记并在此停止。你想要哪个？"律师决定是否接受较低可信度的来源；skill 不为他们决定。

**来源归因。** 用引用来源标记大纲中的每个规则引用、案例引用和权威：`[国家法律法规数据库]`、`[最高人民法院]`、`[元典]`、`[裁判文书网]`、`[人民法院案例库]`、`[北大法宝]`、`[威科先行]`，用于从法律研究连接器检索的引用；`[web search — verify]` 用于网络搜索引用；`[模型知识——需验证]` 用于从训练数据回忆的引用；`[用户提供]` 用于合伙人或资深律师提供的引用。

### 步骤 2：提取相关文件

从证据材料（如已连接：证据管理平台或本地文件系统）：

- 证人撰写的文件
- 发送给证人或从证人收到的文件
- 按姓名提及证人的文件
- 证人在场的会议记录和日历条目

按日期组织。标记热门文件——对理论最重要的那些。

### 步骤 3：构建主题

每个主题是你想要确立或探索的事情。按理论组织：

**背景（始终第一个——在证人防御之前锁定无争议事实）：**
- 角色、任职时间、职责
- 汇报结构
- 他们如何与关键人物互动

**有利事实（在对抗之前锁定它们）：**
- 来自 `~/.claude/plugins/config/claude-for-legal/litigation-legal/CLAUDE.md` 的对我们有利的关键事实，该证人能确立的
- 支持我们理论的文件，由该证人撰写或收到的

**不利事实（用文件对抗）：**
- 无论如何该证人都会被问到的对我们不利的事实——先获取你的版本
- 不利的文件——了解证人会如何解释它们

**质证材料（如果证人陈述与记录矛盾）：**
- 先前不一致的陈述（来自文件、之前陈述书）
- 与你预期他们所说相矛盾的文件

**转折事实：**
- 确立（或削弱）案件所依赖的事实的问题序列
- 这是最精心构建的部分。问题形式遵循步骤 1a 的证人姿态：不利时紧密封闭式确认，友好时受控开放，中立时混合。不要默认为一种模式。

## 此 skill 不做什么

- 进行庭审。大纲是地图；律师驾驶。
- 预测证人会说什么。它为可能的答案做准备，但证人会带来意外。
- 为证人起草其本人未陈述的内容。询问提示是给证人的；证人自己陈述。
- 决定现场要问什么。跟进是律师在法庭上的判断。
