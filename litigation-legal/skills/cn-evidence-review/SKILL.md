# /cn-evidence-review

1. 加载 `~/.claude/plugins/config/claude-for-legal/litigation-legal/CLAUDE.md` → 审查协议、证据清单格式。
2. 遵循以下工作流程和参考规范。
3. 对每份证据：明显充分/存疑需律师判断/明显不足。标注理由。
4. 输出：待证事实清单 + 经审查的证据清单（含三性评估、证明力分级、缺口标注）+ 补充建议。律师审查所有标注后方可用于庭审。

# 中国民事诉讼证据审查工具

## 证据材料使用限制

- **民事诉讼：** 依《民事诉讼法》第68条及《最高人民法院关于民事诉讼证据的若干规定》，通过法定程序获取的证据材料，仅可用于本案诉讼目的，不得用于其他案件或商业目的。
- **行政诉讼：** 通过政府信息公开或行政复议程序获取的材料，使用目的须与申请目的一致。
- **刑事案件辩护材料：** 辩护律师依法查阅的卷宗材料受《律师法》第38条保密义务约束，不得在本案以外使用。
- **仲裁程序：** 依各仲裁机构规则（CIETAC、上仲等），通过仲裁程序取得的证据材料通常负有保密义务，使用范围须符合仲裁协议约定。

## 目的

证据审查有三类材料：明显充分、明显不足、以及需要专业判断的中间地带。本工具完成前两类的分类，使律师的精力集中于第三类。

**这是初步审查。律师审查所有标注。无例外。**

## 引用核实——出处与覆盖范围

当本工具引用法条、司法解释或裁判规则支撑证据评估结论时，适用以下两项规则：

**出处必须支撑完整命题。** 如果审查引用一条法条支撑多要素命题，须核实该条款覆盖每个要素。若仅覆盖部分，应拆分引用或缩窄命题。一处仅支撑部分主张的引用，会在质证时被对方律师指出并动摇整个论点——这是"引用错位"失败模式。

**提取全部引用后再核查。** 如对审查报告本身或相关证据进行引用核查：
1. **第一轮：提取。** 列出全部引用（法条、司法解释、案例、规定）。报告数量："发现 [N] 处引用。"
2. **第二轮：核查。** 逐一对照原文。不得抽样，不得中途停止。
3. **报告覆盖率。** "[M] 处中已核查 [N] 处。[K] 处无法获取——须人工核实。[J] 处已确认。[I] 处标注为疑似错误引用。[H] 处标注为引用错位。"
4. **无法获取原文时，写"未能核查"，绝不写"已确认"。**

## 第零步：研究法院证据规则

**在审查任何证据前，先研究本案法院的证据规则要求：**

适用的核心规范包括《最高人民法院关于民事诉讼证据的若干规定》（2019年修订版）、《民事诉讼法》第六章、以及具体法院的庭审规程。重点确认：

- 举证期限要求（法院指定或双方约定）
- 电子数据的认定标准
- 鉴定意见的采信规则
- 证人出庭要求
- 书证原件/复印件要求
- 本案适用的证明标准（高度盖然性原则）

**不得无声补充。** 如检索结果稀少或规则不明确，报告并停止，由律师决定是否采用置信度较低的来源。标注 `[存疑——须核实]`，不得自行填补。

**来源归属标注。** 审查报告中每处规则引用均须标注来源：`[元典智库]`、`[中国裁判文书网]`；`[网络检索——须核实]`；`[模型知识——须核实]`；`[user provided]`。标注 `须核实` 的引用应优先核查。不得删除来源标注。

## 三档分类规则

**本工具绝不在主观判断不明确时悄然作出结论。** 对任何存疑情形——证明力边界模糊、三性中任一项存疑、举证责任归属不清、对方质证路径不确定——工具保留证据并加 ⚠️ 标注，交由律师判断。证据不足是单向门（庭审失利后难以弥补）；过度标注是双向门（律师在审查中纠正）。默认偏向双向门。

### 明显充分（✅）——保留，无需律师额外审查

- 书证原件，双方签字/盖章，内容与待证事实直接对应
- 生效判决书、仲裁裁决书、公证书等具有较强证明力的法定文书
- 官方登记机关出具的登记证明（工商登记、不动产登记、车辆登记等）
- 己方当事人陈述，与其他证据相互印证，无明显矛盾
- 经公证或可信时间戳认证的电子数据

### 存疑——保留并标注（✅ + ⚠️）

任何不能明确归入✅或❌的证据，默认归入此类。工具不代替律师作主观判断。常见情形：

- **复印件/扫描件：** 原件是否可调取？对方是否会对复印件提出真实性异议？
- **电子数据未经公证：** 微信聊天记录、电子邮件、截图——真实性是否可通过其他证据印证？是否需要申请电子数据鉴定？
- **证人证言：** 证人与当事方是否存在利害关系？是否需要证人出庭？证言内容是否与书证存在矛盾？
- **间接证据：** 证明力取决于与其他证据的组合关系——单独评估意义有限，须结合整体证据链判断。
- **鉴定意见：** 鉴定机构资质是否符合要求？鉴定程序是否合规？鉴定结论是否存在争议空间？
- **举证责任归属不清：** 本项事实应由哪方举证？是否存在法定举证责任倒置情形？
- **证据取得方式存疑：** 取得方式是否合法？是否存在侵犯对方隐私或违反法律强制性规定的风险？

每个 ⚠️ 标注须记明：具体存疑问题 + 支持证明力的因素 + 削弱证明力的因素 + 律师需要作出的具体判断。

### 明显不足（❌）——记录建议，不自动排除

仅用于无争议的明显情形。输出仍记录评估理由，律师确认后决定是否排除或补强。

- 证据内容与待证事实无关联（关联性缺失）
- 明显伪造或变造的痕迹
- 取得方式明显违法（如非法录音、侵入他人设备取得的数据）
- 法律条文、类案裁判文书——不能作为证据使用
- 仅能证明对方主张的证据（对己方不利证据，须单独评估是否主动提交）

如某项❌判断存在任何接近性——证据虽有瑕疵但可能通过补强修复，或法律条文虽不能单独作证据但可与其他材料结合——归入存疑类并标注。

## 工作流程

### 第一步：待证事实梳理

**在审查任何具体证据前，先确定本案待证事实。**

待证事实是指：与争议焦点直接相关、尚存争议、且需要当事人举证证明的事实。

**不属于待证事实的情形：**
- 双方无争议的事实
- 已被生效文书（判决书、调解书、公证书）确认的事实
- 法律适用问题（法律的理解和适用由法院职权处理，不需举证）
- 众所周知的事实

**待证事实的来源：** 结合诉求主张人的核心诉求与争议焦点分析提炼，每项待证事实须明确、具体，可与具体证据形成对应关系。

**输出格式：**

**（一）待证事实**
待证事实1：[具体描述，与争议焦点直接对应]
待证事实2：[具体描述]
……

|---|---|
| 证据名称 | |

缺少字段 → 标注要求补充后再作实质审查。

### 第三步：逐一审查

对每份证据：

```
证据 [编号]（[证据名称]）：[✅ 明显充分 | ✅ + ⚠️ 存疑保留 | ❌ 明显不足（建议）]
三性评估：
  真实性：[评估结论及理由]
  合法性：[评估结论及理由]
  关联性：[评估结论及理由]
证明力：[强 / 中 / 弱 / 无]
对应待证事实：[待证事实X；待证事实Y]
[如为 ✅（无标注）：一行理由]
[如为 ✅ + ⚠️：保留标注；律师需要回答的具体问题；支持证明力的因素；削弱证明力的因素]
[如为 ❌：一行理由——但不自动排除，律师确认后操作]
```

**三性评估标准：**

- **真实性：** 证据是否真实，是否存在伪造、变造；原件与复印件是否一致；电子数据是否经过公证或可信时间戳认证；形成时间是否与案件事实吻合
- **合法性：** 证据取得方式是否合法；是否侵犯他人合法权益；是否违反法律禁止性规定；是否符合法定形式要求
- **关联性：** 证据是否与待证事实存在内在逻辑联系；是否能直接或间接证明待证事实；证明方向是否正确（支持己方还是对方）

**证明力分级：**

| 等级 | 含义 | 典型情形 |
|---|---|---|
| 强 | 单独可完成证明 | 原件书证、生效判决、官方登记文件 |
| 中 | 需与其他证据组合 | 复印件、证人证言、间接证据 |
| 弱 | 辅助印证作用 | 单方陈述、未经认证的电子数据 |
| 无 | 不具证明力 | 与待证事实无关联、取得方式违法 |

### 第四步：举证责任分析

对每项待证事实，明确举证责任归属：

- **谁主张谁举证（一般规则）：** 《民事诉讼法》第67条——当事人对自己提出的主张，有责任提供证据。
- **法定举证责任倒置情形：**
  - 产品责任：生产者须证明产品不存在缺陷（《产品质量法》第41条）
  - 医疗损害：医疗机构须证明诊疗行为与损害后果不存在因果关系（《民法典》第1222条）
  - 环境污染：污染者须证明不存在因果关系（《民法典》第1230条）
  - 劳动争议：用人单位须就解除劳动合同的事实理由承担举证责任（《劳动争议调解仲裁法》第6条）
  - 高度危险责任：受害人须证明损害系高度危险作业造成（《民法典》第1236条但书）

对举证责任归属不明确的情形，标注 ⚠️，由律师确认。

### 第五步：证据缺口识别与补充建议

对每项待证事实，评估现有证据是否足以完成证明，识别缺口并提出补充建议。

**缺口类型：**

| 缺口类型 | 说明 | 处理方向 |
|---|---|---|
| 完全缺口 | 某项待证事实完全无证据支撑 | 优先补充，否则该事实无法完成证明 |
| 证明力不足 | 有证据但证明力弱（如仅有复印件） | 寻找原件或补充其他证据印证 |
| 证据链断裂 | 间接证据链条中缺少关键环节 | 补充中间环节证据 |
| 形式瑕疵 | 证据内容充分但形式存在问题 | 申请补正、公证或重新取证 |

**补充建议应说明：**
- 建议收集的证据名称
- 证据来源（当事人自行收集/申请法院调查令/申请鉴定/申请证人出庭）
- 该证据对应哪项待证事实
- 收集难度评估

**举证期限硬约束提示：**
> ⚠️ 以下证据尚未收集，请在举证期限（[日期]，如已知）届满前补充。逾期提交的证据可能被法院不予采纳（《最高人民法院关于民事诉讼证据的若干规定》第53条）。举证期限届满后申请延期须有正当理由，由法院决定是否准许。

### 第六步：对方质证预判

对每份 ✅ 或 ✅+⚠️ 证据，预判对方可能从三性角度发起的质证攻击方向：

- **真实性攻击：** 对方可能主张证据系伪造/变造；原件不存在；电子数据已被篡改；证人与当事人存在利害关系
- **合法性攻击：** 对方可能主张证据取得方式违法（非法录音、侵入设备）；未经当事人同意
- **关联性攻击：** 对方可能主张证据与待证事实无内在逻辑联系；证明的是其他事实而非争议焦点

对每个预判的质证方向，说明应对策略（补充印证证据/申请鉴定/准备质证意见）。

### 第七步：规律性标注

跨全部证据识别重复模式：

- **同类证据集中存疑：** 如所有电子数据均未经公证——一个决定解决多个标注
- **证据链整体薄弱：** 多项待证事实均依赖间接证据，整体证明力不足
- **形式问题集中：** 多份证据为复印件——统一申请原件核对或公证
- **举证期限风险：** 待收集证据数量较多，需优先排序

## 本工具明确不做什么

- **不作近似判断。** ⚠️ 意味着"人来决定"。对任何主观判断（证明力边界、举证责任归属、对方质证成功概率），工具保留标注并交由律师。
- **不排除证据。** ❌ 是记录给律师的建议，不是对证据清单执行的操作。
- **不替代律师的质证策略判断。** 对方质证预判是线索，不是结论；质证意见的起草是另一项工作。
- **不保证 ✅ 判断的最终正确性。** 律师对证据清单负责。这是初步审查。
- **不捏造法条编号或案例。** 评估中引用的规则须来源于相关材料或已知文献；不确定时标注 `[模型知识——须核实]`。
