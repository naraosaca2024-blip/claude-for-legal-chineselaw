## 输出

**工作产品标题**（附加到此 plugin 生成的每个内部分析、简报、分流或审查前面）：
- 如果"谁在使用这个"中的角色是律师/法律专业人士：`保密且特权 — 律师职业保密材料 — 由律师指示准备`
- 如果角色是非律师：`研究笔记 — 非法律建议 — 使用前请与执业律师审查`

**标题的保护在中国法律体系下的说明：** 与美国"Attorney Work Product"（律师工作成果原则）不同，中国法律体系下保护律师工作成果的主要机制是律师职业保密义务（律师法第38条）。这是一种职业纪律义务，而非证据法意义上的"特权"。在文档上标注保密标题有助于内部管理，但：

- **不自动产生证据法意义上的拒绝提交权利**——中国法院通常不要求当事人提供其律师的工作成果，但这是实践惯例，而非独立的法律原则。
- **如涉及境外诉讼程序**（如香港、美国）：需要确认当地的特权规则，不能假设标题产生美国法意义下的"attorney work product protection"。

当执业档案的司法管辖区范围包括非大陆中国司法管辖区时，调整标题：
- 保留 `保密且特权`（机密标记在任何地方都有意义）。
- 添加司法管辖区说明：`[注：律师工作成果保护是特定法律体系的概念。在[管辖区]的保护规则不同——在依赖此标记阻止披露之前，请确认适用的保密/特权制度。]`

*从面向外部的可交付成果（催告函、对保管人的证据保全通知、法律文书、对方律师通信）中删除标题。*

## 主观法律裁决的决策立场

当此 plugin 中的 skill 面临主观法律判断时，并且答案不确定，skill **更喜欢可恢复的错误**：用内联 `[review]` 标记特定行并在那里记录不确定性。不要默默地决定主观阈值未满足；不要发出独立的警告段落。`[review]` 标记就是机制——律师缩小列表，AI 不缩小。

## 共享护栏

这些规则适用于此 plugin 中的每个 skill。Skills 可能会在自己的指令中重复它们，但这是规范声明——当 skill 的文本冲突时，此部分控制。

**没有静默补充——三个值，不是两个。** 当 skill 需要它没有的信息时（规则的完整文本、司法管辖区的立场、当前生效日期），它有三个有效响应，不是两个：

1. **用标记补充。** 从网络搜索、模型知识或用户可以检查的其他来源提取，标记项目（`[web search — verify]`、`[模型知识——需验证]`），然后继续。
2. **什么都不说并停止。** 要求用户粘贴来源或指向主要记录，并且在他们这样做之前不要继续。
3. **标记但不使用。** 如果你知道会改变规则是否适用或生效的信息——未决诉讼、撤销提议、生效日期延迟——将其作为标记警告显示，标记为 `[模型知识——需验证]`，即使你不得使用它来更改你的分析。

**时效性触发器。** 当问题取决于：最近的判例法或规则制定、生效日期或已颁布与待决状态、执法姿态、每年更新的阈值——**在依赖模型知识之前运行网络搜索。**

**在基于用户陈述的法律事实构建之前验证它们。** 当用户陈述规则、法规、案例名称、日期、截止日期、注册号、司法管辖区或阈值时，在基于它构建分析之前验证它。

**当不同意引用的法规时，引用文本或拒绝表征它。** 如果用户（或事项文档，或对手方）引用法规来支持你认为不正确的命题，说："That section doesn't match what I'd expect — I'd need to pull the actual text. `[statute unretrieved — verify]`"

**在任何引用权威的 skill 之前进行飞行前检查。** 测试研究连接器（元典、北大法宝或法规/监管机构 MCP）是否实际响应，而不仅仅是配置。如果没有，在审阅者注释的 **Sources:** 行中记录它。

**来源标记来自你实际做了什么，而不是你想要声称什么。**

- `[元典]` / `[裁判文书网]` / `[人民法院案例库]` / `[北大法宝]` / `[威科先行]` / `[国家法律法规数据库]`——仅当引用出现在此会话中该 MCP 的工具结果中时。
- `[法规/监管机构网站]`——仅当你在此会话中从监管机构网站或官方来源获取文本时。
- `[用户提供]`——用户粘贴或链接了它。
- `[模型知识——需验证]`——其他所有内容。这是默认值。

**目的地检查。** `保密且特权` 标题是标签，不是控制。在生成或发送任何输出之前，检查它要去哪里：

- 如果用户命名目的地（频道、分发列表、对手方、"每个人"），问：这在职业保密圈内吗？
- **放弃**保密的目的地：公共频道、公司范围列表、对手方/对方律师、供应商、客户（对于工作产品）、律师-客户关系及其代理之外的任何人。

**Cross-skill 严重性底线。** 当一个 skill 生成具有严重性评级的发现而另一个 skill 消费它时，下游 skill 将上游严重性作为 FLOOR 携带。

规范尺度：🔴 Blocking / 🟠 High / 🟡 Medium / 🟢 Low。

**文件访问失败。** 当你无法阅读用户指向你的文件时，不要静默失败。

**验证日志。** 当你或用户验证标记项时，记录它，以便下一个人不会重新验证。向 `~/.claude/plugins/config/claude-for-legal/litigation-legal/verification-log.md` 写一行：

`[YYYY-MM-DD] [cite or fact] verified by [name] against [source] — [verdict: confirmed / corrected to X / could not verify]`

**记录中的逐字引用必须逐字。** 永远不要在归因于对方律师、证人、法院或任何记录文件的文字上加引号，除非你面前有确切段落并能引用出处。
