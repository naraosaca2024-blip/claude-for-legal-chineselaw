# 监管信息源目录（中国法版）

本文件提供 `reg-feed-watcher`（监管动态监测）的整理过的信息源目录，按类别组织。**用于覆盖核查与建议补源**，不是必须接入的全集。

**注**：当前为 v1 框架版。详细的源 URL、API 端点、聚合服务清单将在与团队对齐元典开放平台覆盖范围后逐步完善。可通过 `cold-start-interview --redo` 让用户根据实际工作中遇到的具体监管/协会补充。

**重要：所有部委栏目页 URL 必须在 cold-start 时通过 `WebSearch` + 浏览器访问验证当前有效性后才能登记到 CLAUDE.md。禁止直接硬编码或照搬历史 URL。政务网站改版频繁，静态列表半年内大概率失效。**

**如何阅读本目录：**
- **格式** — 信息源返回的内容：元典 MCP（结构化，最佳）、栏目页（半结构化，需轮询）、RSS/Atom（半结构化）、HTML 页面（需要爬取或变更检测）
- **层级** — *一手* 指监管机构本身；*二级* 指总结主要来源的评论员、聚合器或律所
- **认证** — 公开表示开放；API Key 表示需要注册；付费表示订阅制
- **备注** — 任何注意事项（SPA/动态渲染需求、改版风险、发现步骤）

标有 ⚠️ 的信息源已被用户或监管机构报告为不可靠或有技术访问限制——配置前请验证。

## 2. 元典开放平台 MCP（首选检索来源）

| 源 | 类型 | 说明 |
|---|---|---|
| 元典开放平台 | API / MCP | 本插件唯一推荐的法律检索集成。法规检索 + 案例检索 + 企业信息接口；结构化数据，引用核验有据。配置 yuandian MCP 后优先使用。标签 `[元典法规]` / `[元典案例]` |

**元典作为第 1 级信息源的优势**：
- 已结构化数据（标题、文号、发布机关、生效日期等字段化）
- 覆盖中央层级法律/行政法规/部门规章/司法解释/指导性案例/行政处罚
- 引用核验有据——元典直接对应一手出处
- 政务网站改版不影响（API 稳定）

## 6. 二级来源 / 聚合源

**警告**：标记为 `[二级来源]` 的源，本插件**强制**要求依赖前核对到一手出处。**不允许**仅凭二级来源升到"始终重要"。

公众号 / 自媒体（v1 走人工录入，不自建抓取）：
- 金融监管研究院、零壹财经、互金通讯社、iCFC 数字金融、数据法盟、隐私护卫队等
- 手动录入路径：通过 reg-feed-watcher 的第 4 级手工录入档粘贴条目

律所专业邮件 / 评论：
- 竞天公诚、金杜、中伦、方达、锦天城等律所有定期"金融监管/数据合规/IP/公司法"等领域简报
- 用户可自行订阅，把感兴趣的条目通过 manual entry 路径粘贴进本技能

北大法宝 / 威科先行（有付费订阅则使用）：
- 可作为元典的补充或替代检索源

## 无信息源的来源（需要网页监控或邮件）

一些重要来源不发布信息源，或其栏目页技术实现困难。监控它们需要以下之一：
- 元典 MCP 集成（推荐，结构化，无 URL 失效问题）
- 栏目页轮询（需 playwright MCP 处理 SPA）
- 邮件通讯转发（需 Gmail/Outlook MCP 集成）
- 通过 reg-feed-watcher 的"手工录入"路径进行手动检查

| 来源 | URL | 备注 |
|---|---|---|
| 多数部委官网 | 见部委表格 | SPA/动态渲染，优先通过元典 MCP 接入 |
| 行业协会会员区 | 各协会官网 | 需会员账号，无法自动抓取 |
| 监管机构约谈通报 | 通过协会转述 | 多数情况下仅邮件或网站公告，需手工录入 |

**金融科技 / 消费金融内部法务团队：**
元典 MCP + 国家法律法规数据库（金管总局、证监会、央行、网信办过滤）+ 中国互联网金融协会 + 中国支付清算协会 + 裁判文书网

## 添加信息源

要添加本目录中没有的信息源：

1. 确定监管主体全称（如"国家金融监督管理总局上海监管局"）
2. 通过 `WebSearch` 搜索"[监管全称] 政策法规 栏目"，获取当前官方域名链接
3. 用浏览器访问验证 URL 可达且含政策列表
4. 验证通过后在用户 CLAUDE.md 的**信息源配置**下添加，含验证日期
5. 如果验证失败，标记为"待补"，降级到元典 MCP + 手工录入，**不猜测 URL**

reg-feed-watcher 在步骤 0 覆盖核查中参考本目录，发现用户关注清单中某类别配置稀薄时呈现建议。本目录**不是**自动接入的清单——接入需要用户在 cold-start 中确认并验证。
