# 监管下发函 6 分法 —— 详细分类规则与处置路径

**对应技能**：`regulatory-incoming-letter`（监管来函处置）
**法律依据**：行政处罚法、行政复议法、行政诉讼法、行政强制法、各监管特别法
**核验来源**：`项目文档/法律备忘录_监管下发函6分法分类核验_2026-05-18.md`

本文档展开 SKILL.md 中 6 分法的详细规则，作为监管动态监测自动分类 + 用户人工核对的参照。

## 第 1 类：行政处罚类

### 法律性质

依《行政处罚法》第 9 条，处罚类型包括：
- 警告 / 通报批评
- 罚款、没收违法所得
- 暂扣许可证、降低资质、吊销许可证
- 限制开展业务、责令停产停业、责令关闭
- 行政拘留（特殊情形）

**输出文书**：《行政处罚决定书》（独立文书，监管落款 + 加盖公章）

### 典型函件名称

- 《行政处罚决定书》〔XX〕XX 号
- 《关于对 XX 公司处以罚款的决定》
- 《暂停 XX 业务的决定》
- 《吊销 XX 许可证的决定》

### 关键短语（自动归类触发词）

- "依据《XXX 法》第 X 条，决定对……作出如下处罚"
- "罚款 XX 万元 / 没收违法所得"
- "责令 XX 整改"（出现在决定书中而非工作函中）
- "处罚事先告知书"（→ 实际是第 4 类，注意区分）

### 时效（**最高优先级**）

| 救济路径 | 法律依据 | 期限 | 起算 |
|---|---|---|---|
| 行政复议 | 行政复议法第 9 条 | **60 日** | 知道行政行为之日起 |
| 行政诉讼 | 行政诉讼法第 46 条 | **6 个月** | 知道或应当知道行政行为之日起 |
| 证监会复议 | 证监会规章 | 通常 **60 日** | 决定书送达之日 |
| 银保监 / 金管总局复议 | 行政复议法 | 60 日 | 同上 |
| 网信办处罚复议 | 行政复议法 | 60 日 | 同上 |

**注**：复议 / 诉讼期间**通常不停止执行**（行政强制法 / 行政诉讼法第 56 条原则），除非提出停止执行申请并被批准。

### 处置路径

1. **当天**：法务紧急研判 + escalation_path.重大业务影响决策（CCO + 法务总监）
2. **3 个工作日内**：决定救济路径（接受 / 复议 / 诉讼）
3. **30 日内**：如复议，准备复议申请书
4. **6 月内**：如诉讼，准备起诉状
5. **同时**：处罚决定书中"责令整改"事项拆为 inspection-task

### 风险点

- 处罚事实认定是否准确（公司有 / 无相关行为）
- 法律依据是否恰当（援引法条是否对应）
- 处罚程序是否合法（事先告知、听证、决定书送达）
- 是否有从轻 / 减轻 / 不予处罚情节未被采纳
- 上市公司 / 持牌机构：是否需要披露（信息披露义务 vs 商业敏感）

## 第 2 类：行政监管措施类（含子类 2a / 2b）

### 法律性质

**非处罚性监管措施**。轻于处罚，但仍可能对公司经营有实质影响。

**法律基础**：
- 证监会：《证券期货市场监督管理措施实施办法》（2026-06-30 起施行，**14 类 + 兜底**，不是网传的"18 类"）
- 银保监 / 金管总局：《现场检查办法》、监管规定中的"监管谈话 / 监管意见书 / 风险提示函"
- 网信办：《互联网信息服务管理办法》、《网络安全法》执法权

### 子类 2a：责令整改型

**典型函件**：
- 《责令限期整改通知书》
- 《警示函》（不发到公开渠道的内部警示）
- 《监管意见书》
- 《风险提示函》

**关键短语**：
- "限 X 日内整改完毕，并将整改情况书面报告本 [局 / 委 / 会]"
- "请贵公司高度重视，认真整改"
- "如再发现类似问题，将依法采取进一步监管措施"

**处置**：限期整改 + 拆为 inspection-task + 上报整改报告

### 子类 2b：现场检查反馈型

**典型函件**：
- 《现场检查意见书》
- 《现场检查反馈通知》

**关键短语**：
- "本 [局] 于 X 月 X 日至 X 日对贵公司进行了现场检查，现将检查发现问题反馈如下"
- "检查发现以下问题：…"
- "请贵公司针对上述问题逐项整改"

**处置**：与 2a 类似，但**现场检查通常涉及多个问题清单**，需要逐项分析根因 + 拆分到多个责任部门

### 时效（适用于 2a 和 2b）

| 救济路径 | 期限 | 注意 |
|---|---|---|
| 行政复议 | **60 日** | 但部分领域认为监管措施"不可诉"（如证券公司收警示函），司法实务有分歧。**保守做法**：先评估可诉性 + 同时保留复议路径 |
| 整改报告提交 | 函件指定（通常 30-60 日） | **硬刚性**，不提交会触发进一步监管 |

### 处置路径

1. 收文当日：法务初判 + escalation_path.首道审核
2. 7 日内：法务总监 + 业务部门联席会议
3. 整改任务拆解为 inspection-task + 派单
4. **回函前 5 个工作日**：上报材料汇总 → 会签 → 提交

### 风险点

- 检查事实是否准确（部分问题可能由检查组对法规理解差异引起）
- 整改是否走形式 vs 真实改造（监管后续可能再次检查验收）
- 整改是否会**牵出其他问题**（自查可能发现新问题，需评估是否主动披露）

## 第 3 类：监管谈话类

### 法律性质

**程序性沟通**，不直接产生权利义务变动。一般认为**单独的监管谈话不可诉**（《行政诉讼法》上的"行政行为"门槛较高）。

但**约谈通报**（谈话后的文字记录）如含责令整改、警示等内容，可能升格为第 2 类处理。

### 典型函件名称

- 《关于约谈贵公司主要负责人的通知》
- 《监管谈话纪要》（谈话后下发）
- 《关于派员参加 XX 座谈会的函》

### 关键短语

- "请贵公司主要负责人 / 法定代表人 / 总经理 / 风险管理负责人 / [职位] 于 X 月 X 日 X 时至 [地点] 参加监管谈话"
- "就近期 [事项] 与贵公司交换意见"
- "应携带 [材料清单]"

### 时效

- **派人到场**：通常 7-14 天准备期
- **谈话纪要回复**（如要求）：通常 7-15 日

### 处置路径

1. **当天**：核实通知真实性（重要 —— 监管假冒诈骗也有）→ 内部沟通
2. **派人选定**：默认派"业务对应分管 + 法务"组合
3. **准备材料**：本次涉及业务的近期数据、合规自查结论、已采取的措施
4. **谈话纪要追踪**：谈话后 X 日内监管可能下发约谈纪要 / 通报，按内容**重新分类**（升格为第 2 类或直接结案）

### 风险点

- 谈话过程被录音 / 录像，发言会被监管援引
- 公司方陈述应当**对外口径一致**（避免不同部门给出矛盾陈述）
- 是否需要外聘律师陪同（普通监管谈话不必，但涉及重大违规调查的谈话建议）

## 第 4 类：程序性通知类 ⚠️ **法定回应窗口极短**

### 法律性质

行政处罚法规定的法定程序文书。**回应窗口极短** —— 错过有不利后果。

### 法律依据

| 文书 | 法律依据 | 法定回应窗口 |
|---|---|---|
| 《行政处罚事先告知书》 | 行政处罚法第 44 条 | 收到后**及时**（无明确法定期限，建议 3 个工作日内陈述申辩） |
| 《听证告知书》 | 行政处罚法第 47 条 | 收到后 **3 个工作日内**申请听证 |
| 《重大行政处罚事先告知书》 | 行政处罚法第 63 条 | 3 个工作日内陈述申辩 + 听证申请 |
| 《调查终结报告告知》 | 部分领域规定 | 视具体规定 |

### 典型函件名称

- 《行政处罚事先告知书》〔XX〕XX 号
- 《听证告知书》
- 《关于陈述申辩权利的告知》

### 关键短语 ⚠️（最重要的识别词）

- "本机关拟作出如下行政处罚…"
- "在收到本告知书 3 个工作日内，有权要求举行听证"
- "如对本告知书认定的违法事实、依据、拟作出的处罚决定有异议，可以在 X 日内提出陈述、申辩"

### 时效（**最紧急档**）

- 听证申请：**3 个工作日内**（行政处罚法第 47 条）
- 陈述申辩：建议 **3-7 个工作日**（具体看告知书指定）
- 错过后果：视为放弃听证 / 陈述申辩权 → 监管按事先告知内容作出正式处罚决定

### 处置路径（**所有动作压缩在 3 个工作日内**）

1. **当天**（收文 0 日）：
   - 法务紧急研判：事实有无、法律依据是否成立、是否有从轻情节
   - escalation_path.重大业务影响决策 立即触发
   - 评估是否启动外聘律师

2. **次日**（收文 1 日）：
   - 决策：是否申请听证？是否陈述申辩？
   - 收集证据材料（合规记录、内部审计报告、培训记录等）

3. **第三日**（收文 2 日）：
   - 起草陈述申辩书 / 听证申请书
   - 法务总监 + CCO 联签

4. **第四日**（收文 3 日）：
   - 提交陈述申辩书 / 听证申请书（**预留半天**应对提交故障）

### 风险点

- 听证 / 陈述申辩是**法定权利**，错过即放弃，监管可能直接作出处罚决定
- 如决定**不申请听证**：应当**有书面记录**（内部决策记录），避免后续被质疑"未行使法定救济"
- 即使最终接受处罚，争取从轻减轻情节（首违 / 主动消除危害 / 配合调查）也有意义

## 第 5 类：专项自查 / 工作函类

### 法律性质

**指导性 / 督导性**，非处罚 / 非措施。但**回复要求是硬性**的。

### 典型函件名称

- 《关于开展 XX 专项自查的工作函》
- 《关于开展 XX 行业排查的通知》
- 《关于做好 XX 工作的通知》
- 《督办函》

### 关键短语

- "请贵公司对照以下 [N] 项内容进行自查"
- "在 X 月 X 日前将自查情况书面报告本机关"
- "重点关注 [X]、[Y]、[Z] 等方面"
- "结合本次自查发现的问题，提出整改方案和落实措施"

### 时效

- 自查 + 上报：函件指定（通常 30-60 日）
- **硬刚性**：错过会触发监管约谈 / 现场检查 / 处罚升级

### 处置路径（标准 7 步法）

1. **收文当日**：录入监管来函处置 + 字段抽取
2. **当周内**：拆解自查事项为 `inspection-task` + 派各业务部门
3. **每周节点**：跟踪进度
4. **回复期前 X 周**：归口部门 / 法务汇总各部门反馈
5. **回复期前 5 个工作日**：生成上报草稿 + 会签
6. **回复期前 1-2 个工作日**：法务总监 + CCO + 法定代表人 / 总经理签发
7. **回复期当日 / 前一日**：提交（按要求方式：密码邮件 / 现场 / 挂号信）

### 风险点

- 自查范围理解错位（"自查"边界要扣紧函件原文，避免主动暴露不在范围内的事项）
- 自查发现新问题：**主动披露 vs 内部整改**的策略评估
- 上报材料措辞：避免过度承诺 / 过度披露 / 与历史立场矛盾

## 第 6 类：监管问询类

### 法律性质

**信息收集行为**。不直接产生权利义务变动，但**回函内容会被监管援引**。

### 典型函件名称

- 《监管问询函》
- 《关于核实 XX 情况的函》
- 《信息核实函》（反洗钱 / 反恐怖融资）
- 《关注函》（深沪交易所对上市公司）

### 关键短语

- "请贵公司就 [事项] 作出说明"
- "请提供 [材料清单]"
- "对照本函要求，于 X 月 X 日前书面回复"

### 时效

- 回函：函件指定（通常 5-15 日，短于自查类）
- **关注函**（交易所）：通常 5-10 个交易日

### 处置路径

1. **收文当日**：法务初判 + 评估问询性质（普通核实 vs 重大事项前置调查）
2. **3 日内**：内部数据收集 + 业务部门反馈
3. **5 日内**：起草回函（脱敏 / 范围控制）
4. **回函期前 1 日**：法务总监 + 业务负责人会签
5. **回函期当日**：提交

### 风险点

- **回函范围**：严格按问询事项回应，**不主动**披露范围外事项
- **数据脱敏**：涉及具体客户的，用编号 / 类别替代实名
- **后续可能升格**：问询函的回函质量决定后续是否升格为约谈 / 处罚

## 综合分类决策树

```
收到监管来函
   ↓
是否含"作出行政处罚决定"等结论性表述？
├── 是 → 第 1 类
└── 否
    │
    是否含"听证 / 陈述申辩 / 拟作出处罚"等程序性表述？
    ├── 是 → 第 4 类（**最紧急 3 工作日**）
    └── 否
        │
        是否含"约谈 / 监管谈话 / 派负责人到场"？
        ├── 是 → 第 3 类
        └── 否
            │
            是否含"现场检查 + 反馈"？
            ├── 是 → 第 2 类（子类 2b）
            └── 否
                │
                是否含"自查 / 专项整改 / 排查"？
                ├── 是 → 第 5 类
                └── 否
                    │
                    是否含"说明 / 反馈 / 提供材料"？
                    ├── 是 → 第 6 类
                    └── 否 → 第 2 类（子类 2a）
```

## 跨类别的共同事项

### 函件真伪核实

中国监管诈骗也有（假冒监管发函要求转账等）。所有收到的"监管函件"应当：

1. **核对发文单位**：通过监管公开电话核实（**不要**用函件中提供的电话）
2. **核对文号**：监管机关函件文号有固定格式（如"沪金监函〔YYYY〕XX 号"）
3. **核对盖章**：扫描件应有清晰公章
4. **任何涉及"转账 / 缴费"的"监管要求"** → 必然是诈骗 → 报警

### 紧急程度排序

```
第 4 类（3 个工作日）> 第 3 类（7-14 天）> 第 1 类（视复议期）> 第 5/6 类（按函件指定）> 第 2 类（按函件指定）
```

### 抄送方处理

如函件**抄送其他监管**（如金管总局抄送地方金融监管局）：

- 该抄送方**通常也会单独跟进**（如要求公司单独向其汇报）
- 上报材料可能需要**抄送对方**
- 后续监管检查可能由抄送方发起
