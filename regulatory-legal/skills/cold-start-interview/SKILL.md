# /cold-start-interview

1. 检查 `~/.claude/plugins/config/claude-for-legal/regulatory-legal/CLAUDE.md`。如果在旧缓存路径 `~/.claude/plugins/cache/claude-for-legal/regulatory-legal/*/CLAUDE.md` 存在填充的 CLAUDE.md（没有 `[PLACEHOLDER]` 标记）但在配置路径不存在，将其复制到配置路径并告诉用户迁移了什么。如果 `--check-integrations`，跳过访谈——仅重新运行第 0 部分"连接了什么"核查，并重写 `~/.claude/plugins/config/claude-for-legal/regulatory-legal/CLAUDE.md` 中的 `## 可用集成` 表格。
2. 使用以下访谈工作流。访谈（第 0 部分优先——角色 + 集成——然后关注清单）：哪些监管机构、政策所在位置、什么重要。
3. 连接政策库目录，索引政策。
4. 写 `~/.claude/plugins/config/claude-for-legal/regulatory-legal/CLAUDE.md`（根据需要创建父目录），含关注清单 + 重要性阈值。

在探测集成时：仅当 MCP 工具调用实际成功时才报告 ✓。已配置但未测试的连接器应标记 ⚪ 并附带单行如何确认。永远不要仅根据 `.mcp.json` 声明报告 ✓——这会误导用户认为某些东西已连接而实际没有。

## 目的

每个监管机构都在持续发布。多数对你不重要。此访谈了解要监视哪些监管机构以及——关键是——这里的"重要"意味着什么，以便监控器产出信号而不是噪音。

## Cold-start 检查

读取 `~/.claude/plugins/config/claude-for-legal/regulatory-legal/CLAUDE.md`：
- **不存在** → 开始访谈。
- **包含 `<!-- SETUP PAUSED AT: -->`** → 欢迎用户并提供从该部分恢复。
- **包含 `[PLACEHOLDER]` 标记但没有暂停注释** → 模板从未完成；提供重新开始或从占位符开始的地方恢复。
- **已填充（无占位符、无暂停注释）** → 已配置；除非 `--redo` 否则跳过。

模板结构位于 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md`——将其用作部分支架。将完成的执业档案写入配置路径，根据需要创建父目录。

如果在旧缓存路径 `~/.claude/plugins/cache/claude-for-legal/regulatory-legal/*/CLAUDE.md` 存在 CLAUDE.md 但在配置路径不存在，则向前复制。

## 检查共享公司档案

查找 `~/.claude/plugins/config/claude-for-legal/company-profile.md`。

- **如果存在：** 读取它。显示单行确认："你是 [姓名]，[执业设置]，在 [公司]，[行业]，在 [司法管辖区]运营。对吗？（或说 'update' 以更改共享档案。）"如果确认，跳过公司问题——直接进入插件特定问题。
- **如果不存在：** 你将是此用户设置的第一个插件。在定向和分支后，询问公司问题并将它们写入共享档案（根据插件根目录中 `references/company-profile-template.md` 的模板），然后继续插件特定问题。告诉用户："我已保存你的公司档案——其他法律插件将读取它并跳过这些问题。"

共享档案中的公司问题（如果存在则不应重新询问）：执业设置、公司名称、行业、你卖什么、规模、司法管辖区、监管机构、风险偏好、升级姓名。插件特定问题（实务手册立场、审查框架、内部风格、督导模型等）保持每个插件。

## 安装范围检查

在定向之前，如果你注意到工作目录在项目内部（而非用户的主目录），标记它。说一次：

> **提醒——看起来此插件可能是项目范围的，这意味着我只能读取 [当前目录] 中的文件。如果你想让我从其他地方（下载、文档、云盘）读取文档，请改为安装用户范围——参见 QUICKSTART.md。你可以继续项目范围，但需要将文件移动到此文件夹。**

要求用户在继续前确认：继续项目范围，或暂停以重新安装用户范围。如果工作目录*是*用户的主目录，静默跳过此检查。

## 访谈开始前

首先显示此前言（3-4 个短行，仅此而已）：

> **`regulatory-legal` 适用于跟踪监管发展、评估内规差异和管理合规义务的人员。** 不是你的领域？`/legal-builder-hub:related-skills-surfacer`。
>
> **2 分钟** 获得你的角色、执业设置和主要监管口径。**15 分钟** 添加完整关注清单、重要性阈值、信息源节奏、政策库索引和评议期来源。
>
> 快速还是完整？（随时使用 `/regulatory-legal:cold-start-interview --full` 升级。）

不要读取用户主目录 `~/CLAUDE.md`、`~/user.md` 或其他个人记忆来预填访谈。唯一的输入是用户键入的答案以及他们指向或粘贴的文档。

## 用户选择快速或完整后

一旦用户选择，向他们定向。用自己的声音覆盖：
- **此插件维护什么：** 你的执业档案（关注清单、重要性阈值、信息源节奏）、内规差异台账、整改事项台账、政策对照归档和评议期日历。
- **此设置做什么：** 了解你实际监视哪些监管机构、"重要"对你意味着什么以及你的政策所在位置，并将其写入插件每次读取的纯文本文件。所有回答的都可以稍后更改。完成后，插件的命令将按用户工作的方式工作，而不是按通用模板的方式。
- **数据来源：** 设置仅根据用户的答案构建新的专业档案。它不读取用户的个人 Claude 历史、其他对话或主目录 CLAUDE.md。如果与此对话早期出现相关内容（例如，用户提到监管机构或其行业），在使用前询问。除非用户键入或批准，否则不会将任何内容折叠到配置中。

**为什么这很重要。** 每个摘要、政策对照、内规差异报告都从此访谈编写的配置读取。通用配置产生通用输出——默认关注清单、默认重要性阈值、以及将每个监管机构发声都当成执法行动的摘要。告诉插件用户实际监视哪些监管机构以及这里的"重要"意味着什么，这是"监管 AI 工具"和"发送信号而不是噪音的工具"之间的区别。答案越具体，摘要将越安静和有用。

## 访谈节奏

- **假设答案存在于某处。** 当问题询问可能写在某处的信息——公司描述、实务手册、升级矩阵、风格指南、手册、司法管辖区列表、事项组合——在要求用户从内存键入之前提示链接或粘贴。"粘贴链接或文档，或给我简短版本"是超过一句话的任何内容的默认询问。让人们重新键入已写内容的访谈官未能完成访谈官的第一项工作。
- **批量大小——计算子部分。** "一次不要问超过 2-3 个问题"意味着 2-3 个*可回答的提示*，计算子部分。一个带有 5 个子部分的问题是 5 个问题。测试：用户是否可以在不滚动的情况下回答？如果问题不适合一个屏幕，则太多。尽可能使用结构化点击问题——它们不需要滚动或键入。

**暂停等待真实答案。** 有些问题有快速的点击答案。其他需要用户键入列表（哪些监管机构）、描述校准判断或指向政策库目录。当问题需要超过快速点击时：
- **提出问题并等待。** 平实地说："这需要键入答案——我会等待。"在他们响应之前不要排队下一个问题。
- **对于上传或路径指针（政策库目录、现有关注清单、信息源 URL）：** "粘贴内容、共享文件路径，或说'暂时跳过'。如果你跳过，我将在你的配置中标记缺口，以便你稍后填充。"然后实际等待。
- **在编写执业档案之前：** 审查访谈。列出跳过或用占位符回答的每个问题。说："在编写你的配置之前，这里仍然开放：[列表]。想现在填充其中任何一个，还是将它们留作占位符？"在编写之前等待答案。
- **永远不要**用静默缺口编写执业档案。每个占位符都应该是用户故意跳过的选择，而不是滚动过去未回答的问题。
- **暂停和恢复。** 提前告诉用户："如果你需要停止，说'pause'（或'stop'、或'让我稍后回到这个'），我将保存你的进度。稍后再次运行 `/regulatory-legal:cold-start-interview`，我将在你离开的地方继续。"当用户暂停时，将部分配置写入 `~/.claude/plugins/config/claude-for-legal/regulatory-legal/CLAUDE.md`，顶部带有 `<!-- SETUP PAUSED AT: [section name] — run /regulatory-legal:cold-start-interview to resume -->` 注释，未回答字段上有 `[PENDING]` 标记（与 `[PLACEHOLDER]` 不同）。当设置重新运行并发现暂停的配置时，欢迎用户："欢迎回来。你在 [section] 暂停。你之前的答案已保存。从我们离开的地方继续，还是重新开始？"不要重新询问已回答的问题。

**在设置中验证用户陈述的法律事实。** 当用户使用具体规则引用、法规编号、案例名称、截止日期、阈值、司法管辖区或注册号回答访谈问题时——并且这是你可以自查的内容——在将其写入配置之前进行检查。如果他们所说的与你的理解或他们粘贴的内容冲突，说出来："`[前提已标——需核验]`"。写入 CLAUDE.md 的错误事实会传播到每个未来输出；在这里捕获是产品中最具影响力的时刻之一。

## 访谈

### 开场

> 我将监视你的监管机构并在有变化时告诉你。但"有变化"每天都会发生。我需要知道对你实际上重要什么，这样我才不会误报。

### 快速开始还是完整设置——分支

用户在前言中选择了快速或完整。分支：

**快速开始路径：** 仅询问第 0 部分（角色、执业设置、集成）和关注清单范围。在其他所有内容上使用 `[DEFAULT]` 标记编写配置。关闭："完成。你现在可以开始使用命令了。我对重要性阈值、摘要节奏和政策库结构使用了合理的默认值。当技能的输出感觉不对时，通常是你应该调整的默认值——它会告诉你哪个。随时运行 `/regulatory-legal:cold-start-interview --full` 进行整个访谈，或 `/regulatory-legal:cold-start-interview --redo <section>` 重新做一部分。"

**完整设置路径：** 下面的现有访谈流程。

### 第 0 部分：谁在使用此，连接了什么

在进入监管细节之前的两个快速问题。这些塑造插件如何工作，而不是它能做什么。

#### 谁在使用此？

> 谁将日常使用此插件？（这填充每个技能的工作产品标题和输出框架——持证律师获得"律师工作材料"标识，企业法务获得"内部合规分析"，业务部门人员获得"业务部门合规材料"；都包含面向监管步骤前的审核 checkpoint。）
>
> 1. **律师或法律专业人士** — 律所律师、企业内部持证律师、法律顾问；有律师执业证
> 2. **企业法务（无律师证）** — 法务专员、法务经理、合规专员；有法律背景但无律师证
> 3. **业务部门人员** — 产品、销售、风控、客服等；处理业务但需要法务支持

如果答案是 2 或 3，说一次（不要在每个输出上重复）：

> 你可以使用这里的每个功能——研究、审查、起草、跟踪。两件事在我的工作方式中改变：
>
> 1. **我会将输出框架化为供律师/法务复核的资料，而不是终审结论。** 而不是"绿灯——签署它"，你将获得"这是我发现的，以及签署前要问的问题。"这比你不能确定的绿灯更有用。
> 2. **我会在具有法律后果的步骤之前暂停**——签署合同、终止某人、发送要求函、提交材料、给产品上线放行、回应监管机构。我会询问你是否与律师/法务审查过，并整理简短简报以便与他们的对话更快。
>
> 这不是免责声明。这是插件知道它擅长什么——研究、组织、结构——与关于你具体情况的法律执业判断之间的区别，工具无法替代后者。在正确时刻律师的几小时时间通常比犯错便宜。

如果答案是 3，添加：

> 如果你需要找到律师：律师协会的引荐服务是最快的起点（上海市律师协会、北京市律师协会等地方律协；中华全国律师协会有指引）。法学院诊所、法律援助中心面向部分场景。中小企业可以问当地工商联或行业协会推荐。

#### 连接了什么？

> 此插件可以与以下内容一起工作：法律检索（元典开放平台 MCP）、文档存储（飞书云盘、钉钉云盘、腾讯文档、百度网盘、坚果云）、DM 通知（飞书、钉钉、企业微信）。让我检查你配置了哪些连接器——需要它们的功能将工作，没有它们的功能将优雅地回退到手动而不是静默失败。

**检查实际连接了什么，而不是配置了什么。** `.mcp.json` 中列出的连接器是*可用的*。实际响应的连接器是*已连接的*。这些是不同的，混淆它们会破坏信任。对于此插件使用的每个连接器：
- 如果你可以测试连接（调用简单的 MCP 工具如列表或搜索），仅在成功响应时报告 ✓。
- 如果你无法测试（无法从这里探测），报告 ⚪ "已配置但未验证——打开你的 MCP 设置以确认"并附带单行如何操作。
- 永远不要仅根据配置报告 ✓。

国家法律法规数据库（flk.npc.gov.cn）和国务院政策文件库（gov.cn）是公开网页源，始终可用，不需要 MCP 连接器。

对于显示为未连接的连接器，告诉用户如何连接。示例措辞："元典开放平台未连接。需要在 `.mcp.json` 配置 API key。本插件没有它也能跑——政府网 + 部委公开栏目 + 手工粘贴覆盖基础场景——但连上能加深检索深度和引用核验。"

然后以这种形式报告发现：

> - ✓ [集成] — 已连接（已测试）
> - ⚪ [集成] — 已配置但未验证。打开你的 MCP 设置以确认。
> - ✗ [集成] — 未找到。[功能] 将回退到 [手动替代品]。[如何连接。]

你不需要所有这些。核心功能仅使用公开网页源（国家法律法规数据库、部委公开栏目）和文件访问即可工作。元典加深度；手工粘贴始终有效。如果你稍后设置某些内容，重新运行 `/regulatory-legal:cold-start-interview --check-integrations`。

#### 写入配置

在配置的第一部分之后立即编写 `## 谁在使用这个`、`## 可用集成` 和 `## 输出` 部分，根据模板。`## 输出` 部分已存在——合并到其中，以便工作产品标题取决于角色。

#### 执业设置

> 在关注清单之前还有一个快速问题：
>
> 执业环境是什么？（这塑造内规差异响应流程和升级条目——企业内部法务问法务总监路由，私人执业问审批链 + 计费阈值，政府/援助/诊所问督导结构。）
>
> - **私人执业（独立律师/小所，无层级）** — 我将跳过审批链问题并询问你何时引入外部律师或同事进行第二意见。
> - **中型/大型律所** — 我将询问你的审批链、计费阈值以及谁在你之上签署。
> - **企业内部法务** — 我将询问你的升级路径、法务总监/CCO 是谁以及何时事情转到业务侧。
> - **政府/法律援助/诊所** — 我将询问你的督导结构以及对你执业的任何限制。
> - **我的执业不符合其中任何一种** — 说出来。我会适配。

**不符合框的执业。** 如果用户的执业不匹配上述选项（涉外仲裁、国际公法、法庭之友意见类、学术咨询、公益律师组、行业协会、军事司法、海事，或任何标准类别假定的任何其他内容），提供："听起来你的执业不符合我的通常类别。用你自己的语言告诉我——你做什么、为谁、什么辖区和论坛、工作是什么样的——我将从中构建你的档案，而不是强迫你进入不合适的框。我将跳过或调整不适用的问题。"然后从自由形式描述构建档案，标记哪些模板字段已填充、调整或留空因为它们不适用。从强制配合构建的档案比从实际真实内容构建的稀疏档案更差。

用这个来塑造内规差异响应流程和执业档案中的升级条目：
- **私人执业（独立律师/小所，无层级）：** 跳过升级链问题。重新构建：而不是"谁批准显著不符合项目的响应"，询问"何时你引入外部律师或同事来给第二意见"。执业档案的升级字段映射到*咨询*而非*路由审批*。
- **中型/大型律所：** 询问审批链、计费阈值以及谁在用户之上签署。
- **企业内部法务：** 询问升级路径、法务总监/CCO 是谁以及何时事情转到业务侧。
- **政府/法律援助/诊所：** 替换该设置中使用的督导链（督导律师、主任、督导委员会）。保持结构，重新标记角色。

然后用大白话问升级问题：

### 第 4 部分：信息源（2-3 分钟）

公开网页源是基线——每个团队无论订阅如何都获得监控。
付费源为有订阅的团队添加深度层。

**步骤 1：为命名关注清单映射公开源（需联网搜索确认）**

国家法律法规数据库（flk.npc.gov.cn）和国务院政策文件库（gov.cn）是中央层级监管的稳定一手出处，URL 变动极少——对这两处直接登记默认源。

**其他政务网站（各部委、地方监管、行业协会）的栏目页 URL 必须联网搜索确认当前有效地址后再登记**，禁止直接硬编码或照搬历史 URL。政务网站改版频繁，静态列表半年内大概率失效。

> 对关注清单上每个监管机构，我帮你搜索并验证当前有效的栏目页或列表页 URL（不是首页——列表页才能识别新增）。验证通过的才登记到配置中。验证失败的标记为"待补"，先用元典 MCP + 手工录入推进。

**步骤 2：询问付费订阅（附加，非必需）**

- 元典开放平台订阅？哪些提醒配置？
- 其他法律检索订阅（北大法宝、威科先行等）？

如果是：配置为公开源之上的深度层。如果否：公开源足以继续。

**步骤 3：手工录入降级**

> 如果你从公众号、邮件订阅、外部律师那看到任何想跑过系统的内容——只需粘贴它，我将与你的政策进行比对并跟踪任何内规差异。为此工作，你不需要信息源订阅。

在配置中记录启用了手工录入。

**步骤 4：评议期跟踪** *（这填充 `/regulatory-legal:comments`——评议期日历记录截止日期并在评议窗口打开时呈现决策。）*

> 当我看到来自你关注清单的征求意见稿时，我将自动记录评议截止日期。你想让我标记这些，以便你可以决定是否提交意见吗？

如果是：启用评议台账（`comment-tracker.yaml`）。在配置中记录评议决策的默认所有者。

## 你的执业档案会学

编写执业档案后，用此注释关闭：

> **你的执业档案会学。** 当你使用插件时，它会变得更好：
>
> - 当技能的输出感觉不对时，通常是某个立场要调整。输出会告诉你哪一个。
> - `reg-change-monitor` 代理监视监管信息源；当更改匹配你政策库中的某些内容时，它会标记它以进行内规差异核查。

> - 运行 `/regulatory-legal:cold-start-interview --redo <section>` 重新访谈一部分，或直接编辑配置文件。
>
> 十分钟的设置让你获得可工作的档案。一个月的使用让你获得读起来像你自己写的档案。
