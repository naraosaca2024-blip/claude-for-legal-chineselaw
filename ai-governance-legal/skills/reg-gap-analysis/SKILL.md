# /reg-gap-analysis（中国 AI 法规合规差距分析）

1. 读取执业档案。确认监管足迹和用例登记册已填充。
2. 使用以下框架。
3. 范围确认：此法规是否适用？（管辖权/主体类型/业务类型/规模门槛）
4. 提取法规要求，与执业档案中的当前状态进行比较。
5. 输出：优先化差距 + 带负责人和截止日期的整改计划。
6. 保存为带日期的文档。即使"无差距"也要记录。

```
/ai-governance-legal:reg-gap-analysis "生成式人工智能服务管理暂行办法"
/ai-governance-legal:reg-gap-analysis "智能体规范应用与创新发展实施意见"
/ai-governance-legal:reg-gap-analysis "人工智能拟人化互动服务管理暂行办法"
```

## 步骤 1：范围确认

1. **管辖权**：法规适用境内运营还是境外服务商？
2. **主体类型**：法规针对构建者/部署者/两者？
3. **业务类型门槛**：是否有特定业务要求（如"向公众提供"、"持续性情感互动"）？
4. **规模门槛**：是否有"具有舆论属性"等量化门槛？
5. **场景豁免**：是否有豁免（如内部使用、非持续性情感互动）？

输出：`适用性结论：[完全适用 / 部分适用 / 不适用 / 需进一步确认]`

---

## 步骤 2：法规要求提取与差距分析

逐条提取义务，按以下类别组织并与执业档案对比：

### A. 事前要求
- 安全评估（上线前）；备案/许可；科技伦理审查

### B. 持续运营义务
- 内容安全（过滤/标识/审核）；数据保护；用户权利；人工审核机制

### C. 监测与应急
- 安全监测；事件报告；定期评估

### D. 文档与记录
- 记录保存；配合监管检查

## 此 skill 不做什么

- 不确定法规是否"真正"适用——适用性判断标记 `[审阅]`，须律师确认。
- 不替代向监管机构提交的正式申报材料。
- 不覆盖中国内地以外的 AI 治理法规。
