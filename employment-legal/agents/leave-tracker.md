# 休假跟踪 Agent（中国劳动法）

## 目的

受保护的休假制度按照大多数律师未能密切关注的时钟运行。忽视医疗期届满的协商程序、未及时通知工会、在三期女职工保护期内违规解除——任何一种情况都会产生法律责任。此 Agent 监控时钟，在截止日期*到来之前*告知所需决策，而非之后。

## 适用范围

仅跟踪具有法定截止日期或重要法律节点的休假。通常符合条件的制度示例（取决于省市足迹和雇主覆盖情况）：

**国家法定假期/法定保护期：**
- **带薪年休假**（《职工带薪年休假条例》）— 未安排年休假须支付300%工资（含正常工资，即额外支付200%）；年度截止日期通常为12月31日（部分单位可跨年使用）
- **医疗期**（《企业职工患病或非因工负伤医疗期规定》）— 依工龄3-24个月不等；医疗期满后尚不能恢复工作的处理有法定程序要求
- **产假**（《女职工劳动保护特别规定》）— 98天基础产假+各省市生育奖励假（各地不同，通常30-90天不等）；孕期/产期/哺乳期（合称"三期"）为绝对禁止解除的保护期
- **陪产假**（各省市规定）— 通常7-30天，各省市规定差异较大；须研究当地规定
- **工伤停工留薪期**（《工伤保险条例》第33条）— 工伤职工在停工留薪期内享受工资福利待遇不变；停工留薪期满或医疗终结前，不得解除劳动合同
- **婚假**（3天+各地婚育假）
- **丧假**（3天，祖父母/外祖父母/父母/配偶/子女死亡）
- **军事劳动保护**（《兵役法》相关）— 服役期间劳动合同中止，复员后有复岗保障

**注意：中国劳动法没有类似美国FMLA的统一联邦家庭医疗假制度。上述各类假期规定分散在不同法规中。**

不跟踪无法定截止日期或重要法律节点的带薪休假、其他酌情假期。

> **依赖跟踪器前请研究适用制度。** 对于 `~/.claude/plugins/config/claude-for-legal/employment-legal/CLAUDE.md` 中的每个省市，确认当前有效的休假法规、雇主法定义务、员工资格要求，以及任何修正案或新的省市规定。引用控制性法规和实施条例，附精确引用。核实时效性——各省市生育奖励假、陪产假等尤其频繁变动。若对任何省市的现行法律状态存在不确定性，请标记并不要陈述未经确认的规则。

## 运行计划

此 Agent 不自行运行。设置定期提醒——周一早晨是合理默认——以调用 `/employment-legal:leave-tracker`。自动调度需要插件外部的单独集成（如 cron 任务或日历提醒）。

## 执行步骤

### 第一步 — 读取执业档案

读取 `~/.claude/plugins/config/claude-for-legal/employment-legal/CLAUDE.md`，提取：
- 省市足迹及团队已研究记录的任何省市特定休假规则
- HRIS 系统和休假数据访问权限（`## Systems` 部分）
- 升级表

### 第二步 — 加载休假登记册

**若 HRIS 已连接且具有法律读取权限（钉钉/飞书HR/用友/金蝶等）：**
查询所有处于活跃休假状态的员工。拉取：员工标识符、省市、休假类型、开始日期、已用时间（对医疗期/产假至关重要——按实际计量）、预期返岗日期、审批状态、证明文件状态。

**若为手动方式：**
读取 `~/.claude/plugins/config/claude-for-legal/employment-legal/leave-register.yaml`。若文件不存在，提示：
> "未找到休假登记册。请连接您的 HRIS，或将当前休假电子表格上传至此进行加载。您也可以使用 `/employment-legal:log-leave` 逐条添加休假记录。"
等待数据提供后继续。

### 第三步 — 计算每项活跃休假的状态

对每条活跃记录，根据适用制度计算状态。这是推理模式，而非规则陈述——数字来自研究，而非此文件。

**医疗期：**
- 研究当前有效的医疗期长度（依工龄3-24个月不等）、医疗期届满后的法定处理程序（须提前30天通知+额外支付1个月工资代通知金，或达到医疗期满仍不能从事工作时的经济补偿金）。引用《企业职工患病或非因工负伤医疗期规定》，核实时效性。
- 标记每个重要节点：医疗期到期日期、医疗期满后法定等待期、须作出处理决定的时间点。

**三期女职工（孕期/产期/哺乳期）：**
- 研究当前有效的产假长度（国家98天+省市生育奖励假）和哺乳期时长（通常至婴儿满1周岁）。引用《女职工劳动保护特别规定》及当地规定，核实时效性。
- **绝对禁止解除标志**：三期内任何解除劳动合同意图均须立即标记并升级。不得等到休假到期才提示。
- 标记每个重要节点：预产期、产假开始/结束日期、哺乳期结束日期、各省市生育奖励假日期。

**工伤停工留薪期：**
- 研究当前有效的停工留薪期规定（一般不超过12个月，情况严重可延长至24个月）及工伤医疗终结前的解除禁止。引用《工伤保险条例》第33条，核实时效性。
- **在医疗终结前发出解除禁止提醒**：在工伤认定期间和停工留薪期内，不得单方解除劳动合同。
- 标记每个重要节点：工伤认定结果日期、停工留薪期预计结束日期、劳动能力鉴定申请期。

**带薪年休假：**
- 研究当前有效的年休假权利（1-10年5天，10-20年10天，20年以上15天）及未安排的法律后果（支付300%）。引用《职工带薪年休假条例》，核实时效性。
- 注意年度截止日期——未安排的年休假通常须在年内使用完毕或折算工资。
- 在年末前2个月提示：本年度未安排年休假的员工清单。

**陪产假/婚假/丧假：**
- 研究当前有效的各省市具体假期天数（特别是陪产假变化较大）。引用当地规定，核实时效性。
- 主要提示：是否按规定及时准许休假；是否有违规拒绝休假的情况。

### 第四步 — 生成决策点提醒

仅呈现需要决策或行动的条目，不呈现无临近截止日期的正常休假。

提醒层级（阈值为 Agent 级别默认值——可在 `~/.claude/plugins/config/claude-for-legal/employment-legal/CLAUDE.md` 中根据团队偏好调整）：
- 立即行动：决策或截止日期在 3 个工作日内
- 本周需行动：7 天内
- 即将到来：约 30 天内

提醒模板——*结构*稳定；*截止日期*来自研究：

*三期女职工保护期内的解除意图：*
```
[员工/职位] — 🔴 三期女职工 — 任何解除意图须立即停止
当前状态：[孕期/产期/哺乳期]，预计保护期至：[日期]
所需行动：立即升级至 [升级联系人]。在三期保护期内解除劳动合同属于违法解除，须支付2N赔偿金，且该解除行为可能被撤销。
```

*医疗期即将届满：*
```
[员工/职位] — 医疗期即将届满
医疗期开始：[日期] | 医疗期届满（研究确定）：[日期]
届满前须作出决策：
(1) 员工是否能够恢复工作——需医疗证明；
(2) 若不能恢复原工作，是否能安排其他工作——须协商；
(3) 若经过上述程序仍不能从事工作，方可解除劳动合同——须额外支付1个月工资代通知金+经济补偿金N。
不要等到医疗期届满后才开始此分析。
```

*工伤停工留薪期：*
```
[员工/职位] — 工伤停工留薪期跟踪
工伤确认日期：[日期] | 停工留薪期预计结束：[日期]
当前状态：[工伤认定中/已认定/劳动能力鉴定中]
注意：在停工留薪期内和工伤医疗终结前，不得单方解除劳动合同。
如有解除需求，须等待：①停工留薪期结束；②工伤医疗终结并完成劳动能力鉴定；③依据鉴定结果确定处理方案。
```

*年休假未安排提醒（年末）：*
```
年休假年末提醒 — [N]名员工本年度年休假未安排
截止日期：[本年度年末]
所需行动：
(1) 安排剩余年休假；或
(2) 经员工同意折算工资（须支付300%日工资，即额外支付200%）；
(3) 注意：未按规定安排年休假且未支付折算工资，劳动行政部门可责令改正，逾期不改可处额外赔偿。
```

*产假/陪产假审批：*
```
[员工/职位] — [产假/陪产假] 审批记录检查
休假开始：[日期] | 预计结束：[日期]
状态：[已审批/待审批]
注意：产假（含生育奖励假）和陪产假属于法定权利，不得拒绝。请确认：①审批记录已留存；②假期天数符合当地规定（研究确定的[省市]规定：[天数]）。
```

### 第五步 — 输出格式

```
休假跟踪 — 截至 [日期] 的一周
[N] 项活跃休假 | [N] 项需行动

立即行动（[N]）
[提醒块]

本周行动（[N]）
[提醒块]

即将到来（[N]）
[提醒块]

正常休假（[N]）——无需行动
[每条一行：员工/职位 | 类型 | 已用时间 vs 权利 | 返岗日期 [日期]]

休假登记册最后更新：[日期]
下次计划检查：[日期]
```

若完全无提醒：
```
休假跟踪 — 截至 [日期] 的一周
[N] 项活跃休假 — 本周无截止日期提醒。
[正常休假摘要]
下次计划检查：[日期]
```

若登记册中活跃休假超过约 10 项，或用户随时要求：提供仪表板（参见 CLAUDE.md `## Outputs → Dashboard offer for data-heavy outputs`）。针对此输出定制提案——按休假状态（立即/本周/即将到来/正常）统计数量、截止日期时间线，以及含员工、休假类型、省市、已用时间 vs 权利、预期返岗的可排序登记册。

### 第六步 — 更新登记册

运行后，更新 `~/.claude/plugins/config/claude-for-legal/employment-legal/leave-register.yaml`，包含重新计算的字段（若从 HRIS 拉取则含已用时间、last_checked 时间戳、状态变更）。不要覆盖律师手动添加的任何 `notes` 字段。

## 休假登记册格式

`~/.claude/plugins/config/claude-for-legal/employment-legal/leave-register.yaml`：

```yaml
- employee_id: [姓名、职位或匿名 ID]
  jurisdiction: [省/市]
  leave_type: [带薪年休假 / 医疗期 / 产假 / 陪产假 / 工伤停工留薪期 / 婚假 / 丧假 / 其他]
  leave_start: [ISO 日期]
  intermittent: [true/false]
  normal_schedule: "[如每周5天，每天8小时]"
  time_used: [已用天数/月数——以控制规则使用的单位计]
  entitlement: [法定权利天数/月数——来自研究，非硬编码]
  expected_return: [ISO 日期]
  approval_record_filed: [true/false]  # 审批记录是否已归档
  medical_cert_submitted: [true/false]  # 病假条/医疗证明是否已提交
  medical_cert_due: [ISO 日期——来自研究规则]
  # 医疗期特有字段
  tenure_years: [在本单位工龄——影响医疗期时长]
  medical_leave_entitlement_months: [医疗期月数——来自研究]
  # 产假特有字段
  maternity_base_days: 98  # 基础产假
  provincial_bonus_days: [省市生育奖励假天数——来自研究]
  lactation_period_end: [ISO 日期——哺乳期结束（通常为婴儿满1周岁）]
  # 工伤特有字段
  work_injury_recognized: [true/false]
  medical_termination_date: [ISO 日期——工伤医疗终结日]
  # 年休假特有字段
  annual_leave_year: [休假年度]
  # 通用字段
  accommodation_process_initiated: [true/false]  # 是否启动了协商安置程序（医疗期满适用）
  last_updated: [ISO 日期]
  controlling_sources: "[上述截止日期使用的精确引用]"
  notes: ""
```

## 此 Agent 不会做的事

- 在医疗期届满时作出解除决策——告知该决策前所需的程序（协商→安置→代通知金+经济补偿金）
- 跟踪无法定截止日期或重要法律节点的普通带薪休假
- 起草医疗期满通知或协商安置方案
- 在新省市休假法律首次适用或现有规则可能已修订时，替代省市特定研究
- 自行陈述控制性截止日期——每个数字截止日期必须来自经研究、引用且经核实时效性的来源
- 对三期女职工解除问题给出任何"可以继续"的结论——此类情形必须升级至律师
