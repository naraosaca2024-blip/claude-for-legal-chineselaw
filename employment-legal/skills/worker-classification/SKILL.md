为省市运行适用的劳动关系认定标准，并标记拟议安排与你试图使用的结构不匹配的地方。仅限前瞻性——对于现有关系，咨询律师。

## 目的

最昂贵的用工分类决定是没有人有意识地做出的那个。有人描述他们想要的（"一个外包人员"），用工开始，两年后事实看起来像是劳动合同关系。此 skill 在拟议安排开始之前运行适用认定标准——并告诉你当你描述的内容与你试图使用的结构不匹配的时候。

**中国劳动法用工形式概览：**

| 用工形式 | 主要特征 | 法律规范 |
|---|---|---|
| 劳动合同关系 | 管理从属性，用人单位指挥、监督 | 《劳动合同法》 |
| 劳务关系 | 平等主体间的劳务合同，无从属性 | 《民法典》合同编 |
| 劳务派遣 | 派遣单位与劳动者签劳动合同，派往用工单位 | 《劳务派遣暂行规定》 |
| 业务外包 | 承接外包的用工主体对劳动者行使管理权 | 合同法+行业规定 |
| 非全日制用工 | 每日不超4小时、每周不超24小时 | 《劳动合同法》第68条 |
| 新就业形态（平台用工） | 平台经济、灵活用工，认定尚有分歧 | 政策指导意见 |

此 skill 教授推理模式。它不陈述法律。每个认定标准、法律引用、要素必须来自适用省市的当前研究。

## 仅限前瞻性硬门——在接收之前运行

**此 skill 分析的是用工开始前的拟议安排。** 在任何实质性接收（步骤 1）之前，询问：

> 这项用工已经开始了吗？该人目前是否在受聘中，或他们是否以这种安排工作了任何时间段（天、周、月或年）？

如果答案是肯定的——用工已以任何形式存在任何持续时间——**停止**。不要进入步骤 1 接收。对现有安排进行分类不是规划练习；它是带有补救影响的负债评估：欠薪（加班费、用餐/休息时间溢价）、未缴纳的社会保险（须补缴并可能受罚）、工伤保险缺口（如被认定劳动关系，用人单位须承担工伤赔偿责任）、劳动合同双倍工资风险（未签书面合同超1个月）。该分析需律师主导，并与补救计划结合。

准确输出此块并等待回应：

> **超出范围——现有安排。**
>
> 此 skill 设计为在用工开始之前分析用工安排，以便用工分类选择指导如何构建合同和运营。你描述的是一个已经存在的安排。对现有用工进行回顾性分析是不同的练习：劳动关系认定风险评估结合补救规划——欠薪风险、社保补缴风险、劳动合同双倍工资风险、工伤责任风险和前瞻性重组。该工作应该由律师主导，并可能需要外部劳动法律师审查，考虑到金额和法律风险。
>
> 建议的下一步：按照你配置的升级表升级（对于回顾性用工分类，通常路由到合规总监/主任合伙人 + 外部劳动法律师）。我已标记此事项进行升级路由。
>
> **如果你仍想为规划目的进行前瞻式分析，说"继续"——但理解：**
>
> - 输出不是补救计划，不应被当作补救计划对待。
> - 输出不覆盖已用工期间的欠薪、社保风险或劳动合同双倍工资风险。
> - 输出不替代此事实模式实际需要的劳动关系认定风险评估。
> - 输出将携带醒目的横幅反映此范围不匹配，后果行动门将要求律师确认后才能将分析视为可靠。
>
> 只有在你使用此 skill 进行前瞻性规划（例如，"如果我们今天从零开始构建，我们应该怎么考虑？"）并且你有补救问题的单独计划时才说"继续"。

**仅在用户明确说"继续"（或等效的用户指令）时才通过此门。** 如果用户选择继续，在此会话中此 skill 的每个输出前加上此横幅：

## 工作流

### 步骤 1 — 信息收集

在单个块中询问以下所有内容。不要逐个问题提问。简要解释你为什么问——律师在理解问题测试什么时回答得更好。

> 为了运行正确的劳动关系认定，我需要详细了解拟议安排。请尽可能多地回答以下内容——画面越完整，分析越准确：
>
> **工作**
> - 此人每天实际会做什么？
> - 这项工作是你公司核心业务的一部分，还是外围的？
>   （例如，互联网公司的软件工程师 = 核心；律所的保洁外包 = 明显外围）
> - 这是一个有明确终点的定义项目，还是持续不确定的工作？
> - 技能有多专业化？此人是否有你团队没有的专业知识？
>
> **管理从属性（核心认定标准）**
> - 谁设定他们的时间和日程——他们还是你？
> - 他们将在哪里工作——你的办公室、他们的地点，或两者？
> - 你会指导他们如何做工作（方法、流程、考核），还是只要求最终结果是什么？
> - 他们会使用你的工牌/门禁/内部系统吗？
> - 他们会参加你公司的内部会议、培训吗？
> - 你的人员会对他们的日常工作进行管理考核吗？
>
> **经济**
> - 他们将如何获得报酬——按时间、按项目收费？发票开具主体是谁（个人还是公司）？
> - 你会提供设备、工具或软件，还是他们使用自己的？
> - 他们为其他公司工作，还是这将是排他的？
> - 他们有自己的商业实体（个体工商户、公司）吗？
>
> **安排**
> - 你想如何构建这个——直接劳动合同、劳务合同、劳务派遣（通过派遣单位）还是业务外包（公司对公司SOW）？
> - 如果是劳务派遣：谁付钱给工人——派遣单位还是你？谁控制日常工作？岗位是否属于"临时性、辅助性、替代性"（三性）？
> - 会有书面合同吗？你有模板吗？
> - 用工大约多长——几周、几个月、超过1年？
> - 他们会和你的正式员工一起做类似的工作吗？
>
> **认定的目的**
> - 认定需要服务于什么法律目的——社会保险缴纳、工伤责任认定、加班费计算、劳动合同解除补偿？不同目的可能影响认定结论。
>
> **省市**
> - 此人将在哪里实际执行工作（工作地点省市）？

在继续之前等待回应。如果律师无法回答某些问题，记录差距——它们影响分析。

### 步骤 2 — 识别适用认定标准

> **在继续之前研究适用标准。** 对于接收中识别的省市，研究当前有效的劳动关系认定标准。中国劳动法通常使用以下认定依据：
>
> **核心依据（国家层面）：**
> - 原劳动和社会保障部《关于确立劳动关系有关事项的通知》（劳社部发〔2005〕12号）：三要素认定——①用工双方具备主体资格；②用人单位依法制定的各项劳动规章制度适用于劳动者，劳动者受用人单位的劳动管理，从事用人单位安排的有报酬的劳动；③劳动者提供的劳动是用人单位业务的组成部分。
> - 辅助证据：工资支付凭证（发放花名册）、社保缴纳记录、工作证/服务证、招聘记录、考勤记录。
>
> **劳务派遣认定：**《劳务派遣暂行规定》（2013年，2016年修订）
> - 三性岗位限制：临时性（存续不超6个月）、辅助性（非主营业务）、替代性（特殊情形下的替代）
> - 用工比例：派遣工数量不超过用工单位用工总量的10%
>
> **"假外包真派遣"认定：** 如果业务外包名称下，实际上由发包单位直接管理和指挥劳动者，可能被认定为劳务派遣而非真正外包。
>
> **新就业形态/平台用工：** 各地正逐步出台指导意见，认定尚有分歧——需研究当地最新规定。
>
> 引用控制性法律条文和规章。注意每条规则的生效日期以及是否最近被修订。如果你对任何省市法律当前状态不确定，标记供律师验证。
>
> **没有静默补充。** 如果研究查询返回很少或没有结果，报告发现的内容并停止。

**来源归因：** `[元典法规]` / `[元典案例]` / `[网络检索 — 需核验]` / `[模型知识 — 需核验]` / `[用户提供]`。

### 步骤 3 — 将研究的标准应用于事实

对于步骤 2 中识别的每个认定标准，将其应用于接收事实。明确为每个要素或审慎评分——不要总结。律师需要看到哪些要素是干净的，哪些是问题。

使用类似下面的结构，但用研究标准中的*要素*填充：

```
认定依据：[认定依据名称，按研究]
目的：[此标准管辖什么——劳动合同关系认定/派遣认定/外包认定]
来源：[精确引用到法规/规章/司法解释]
时效性：[验证截至日期]

| 认定要素 | 接收事实 | 倾向方向/是否满足 |
|---|---|---|
| [研究标准中的要素1，如"主体资格"] | [来自接收] | [倾向劳动关系/倾向劳务/不确定] |
| [要素2，如"管理从属性"] | [来自接收] | [...] |
| [要素3，如"业务组成部分"] | [来自接收] | [...] |

认定标准结构：
[标准如何权衡要素——从研究陈述，而不是记忆]

此标准下的结果：
[倾向劳动合同关系 / 倾向劳务关系 / 倾向派遣关系 / 不确定——争议要素]
```

**关于争议要素的说明。** 一些要素在案例法中争议很大且对事实敏感。明确识别争议要素——不要掩盖它们。

### 步骤 4 — 分类并标记差距

**分类判断**

基于认定结果，陈述此拟议安排最准确的分类：

- **劳动合同关系：** 事实在认定标准下支持劳动合同关系。用人单位须承担劳动法全部义务（社保、加班费、经济补偿金等）。
- **劳务关系（个人）：** 事实在认定标准下支持劳务关系（适用民法典合同编，无劳动法从属关系）。
- **劳务派遣：** 通过持有劳务派遣许可证的派遣单位用工——派遣单位签劳动合同、缴社保；用工单位承担安全生产责任和同工同酬义务。
- **业务外包（真实）：** 发包单位不直接管理劳动者——承包单位自主管理其员工；须确保不构成"假外包真派遣"。
- **非全日制用工：** 每日不超4小时、每周不超24小时，可用口头合同，不缴纳社保中的工伤外险种（工伤除外）。
- **新就业形态/平台用工：** 认定存在分歧——需研究当地最新政策口径并标记争议。
- **不明确/接近判断：** 事实在一个或多个认定标准下两方面都有说服力——陈述哪个标准是问题以及为什么。

**差距分析**

这是最重要的输出。将预期结构与事实实际支持的进行比较：

```
预期结构：[他们说的想要的]
事实暗示的：[研究的标准说这实际是什么]

差距——安排与预期结构不匹配的地方：
🔴 [要素]：[他们描述的] 与 [预期用工形式] 冲突，
   因为 [具体研究标准语言 + 引用]。如果按描述进行用工，这是重大的错误分类风险。
🟡 [要素]：[他们描述的] 在 [标准] 下是较弱点。单独不是淘汰项，
   但与其他因素结合会增加风险。
✅ [要素]：支持 [预期用工形式]。无问题。
```

## 此 skill 不做什么

- 如果错误分类已经发生就补救提供建议。
- 就任何省市自行陈述法律——每个认定标准、要素和豁免必须来自验证的当前研究。
- 在接近判断时替代外部律师——严格认定省市、争议要素和先前审计情况应始终在用工开始前获得人工审查。
