# 冷启动访谈：劳动法律顾问（中国劳动法）

## 目的

中国劳动法到骨子里都是省市的。在上海是正确答案的内容在广东可能有不同的裁判口径。此访谈映射你的足迹——每个有员工的省市和国家——并构建一个知道哪些规则在哪里适用的升级表。

## 冷启动检查

读取 `~/.claude/plugins/config/claude-for-legal/employment-legal/CLAUDE.md`：
- **不存在** → 开始访谈。
- **包含 `<!-- SETUP PAUSED AT: -->`** → 问候用户并提供从该部分恢复的选项。
- **包含 `[PLACEHOLDER]` 标记但没有暂停注释** → 模板从未完成；提供从头开始或从占位符开始处恢复的选项。
- **已填充（无占位符，无暂停注释）** → 已配置；除非 `--redo` 否则跳过。

模板结构位于 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md`——使用它作为章节脚手架。将完成的执业档案写入配置路径，根据需要创建父目录。

## 检查共享公司档案

查找 `~/.claude/plugins/config/claude-for-legal/company-profile.md`。

- **如果存在：** 读取它。显示一行确认："你是 [名称]，[执业设置]，在 [公司]，[行业]，在 [省市/司法管辖区] 运营。对吗？（或说'更新'来更改共享档案。）" 如果确认，跳过公司问题——直接进入插件特定问题。
- **如果不存在：** 你将是此用户设置的第一个插件。在引导和分支之后，询问公司问题并写入共享档案，然后继续插件特定问题。告诉用户："我已保存你的公司档案——其他法律插件将读取它并跳过这些问题。"

属于共享档案的公司问题：执业设置、公司名称、行业、你卖什么、规模、司法管辖区、监管机构、风险偏好、升级名称。插件特定的问题（剧本立场、审查框架、内部风格、督导模式等）保留在每个插件。

## 安装范围检查

在引导之前，如果你注意到工作目录在项目内（不是用户的主目录），标记它：

> **注意——看起来此插件可能是项目范围的，这意味着我只能读取 [当前目录] 中的文件。如果你需要我从其他地方读取文档，改为用户范围安装。你可以继续使用项目范围，但需要将文件移到此文件夹中。**

在继续之前要求用户确认。

## 在访谈开始之前

以分支优先的引导开场。保持 3-4 行简短文字。在其他任何事情之前询问快速或完整。

> **`employment-legal` 适用于处理招聘、解除、调查、休假、规章制度、劳动关系认定和多省市扩张的人。** 不是你的领域？`/legal-builder-hub:related-skills-surfacer`。
>
> **2 分钟**获得你的角色、执业设置和省市足迹（有员工的省市 + 国家），加上解除风险标志、经济补偿金姿态和员工手册政策的工作默认值。**15 分钟**额外添加从先前备忘录提取的真实解除审查触发器和高风险标志、录用函和解除协议模板、省市特定手册补充、劳动关系认定默认值和休假跟踪器集成。
>
> 快速还是完整？（随时可以用 `/cold-start-interview --full` 升级。）

**快速启动路径：** 仅询问第 0 部分（角色、执业设置、集成）和省市足迹。在其余所有内容上用 `[DEFAULT]` 标记写入配置。结束时说："完成。你现在可以开始使用命令了。我对解除风险阈值、经济补偿金姿态和手册政策使用了合理的默认值。当 skill 的输出感觉不对时，那通常是你应该调整的默认值——它会告诉你哪个。随时运行 `/employment-legal:cold-start-interview --full` 做完整访谈，或 `/employment-legal:cold-start-interview --redo <section>` 重新做一个部分。"

**完整设置路径：** 下面的现有访谈流程。

## 用户选择快速或完整之后

给出更完整的引导。一段话，用你自己的语气：

> "此插件维护：你的执业档案（省市足迹、解除标志、员工手册引用）、一个带截止日期警报的休假登记册，以及一个调查案卷文件结构。它学习你实际如何工作——你的执业、你的风险校准、你的内部约定——并将这些写入一个插件每次读取的纯文本文件。你回答的一切都可以稍后更改。"

然后："准备好了吗？先几个快速问题，然后我们深入。"

**为什么这很重要**（如果用户对时间成本有异议时提供）。此插件中的每个命令都读取此访谈写入的配置。通用配置给出通用输出——默认的省市表、默认的高风险解除标志列表、默认的升级矩阵，以及将上海和广州同等对待的审查。告诉插件实际的足迹、实际的招聘和解除触发器以及实际的汇报线，就是"一个劳动法 AI 工具"和"一个知道你的人在哪里以及什么曾经咬过你的工具"之间的区别。

访谈信息仅来自用户输入的答案和他们明确上传的文档。

## 访谈节奏

- **假设答案已经在某处存在。** 当一个问题询问可能已经写好的信息——公司描述、剧本、升级矩阵、员工手册、省市列表——在要求用户凭记忆输入之前提示提供链接或粘贴。
- **批次大小——计算子部分。** 一次永远不要问超过 2-3 个可回答的提示。

**为真实答案暂停。** 一些问题需要上传文档（员工手册、解除备忘录）。当一个问题需要超过快速点击时：
- **提问并等待。** 明确说："这个需要输入——我会等。" 在用户回应之前不要移到下一个问题。
- **对于上传：** "粘贴内容、分享文件路径或说'现在跳过'。如果你跳过，我会在配置中标记差距以便你稍后填充。" 然后实际等待。

**暂停和恢复。** 提前告诉用户："如果你需要停止，说'暂停'，我会保存你的进度。" 当用户暂停时，将部分配置写入顶部带有 `<!-- SETUP PAUSED AT: [部分名称] -->` 注释和未回答字段上的 `[PENDING]` 标记的配置文件。

**在设置中验证用户陈述的法律事实。** 当用户用具体规则引用、法律条文编号、截止日期、阈值、司法管辖区回答访谈问题时——在写入配置之前进行检查。写入 CLAUDE.md 的错误事实会传播到每个未来输出。

## 访谈

### 开场

> 中国劳动法是"看情况"最常成为诚实答案的执业领域。我需要你的地图才能告诉你任何有用的东西——你的人在哪里，你已经处理过什么？

### 第 0 部分：谁在使用，连接了什么

在我们进入劳动特定内容之前三个快速问题。这些决定插件如何工作，而不是它能做什么。

#### 谁在使用？

> 谁会日常使用此插件？（这会填入每个解除备忘录、规章制度草案和调查摘要的工作底稿标题——律师输出获得保密标题，非律师输出获得"研究笔记，与律师审查"标题。）
>
> 1. **律师或法律专业人员** — 律师、律师助理、在律师监督下工作的法律运营。
> 2. **有律师对接渠道的非律师** — 创始人、业务负责人、HR、采购；你有内部或外部律师可以咨询。
> 3. **没有常规律师对接渠道的非律师** — 你自己处理这个。

如果答案是 2 或 3，说一次（不要在每个输出上重复）：

> 你可以使用这里的每个功能——研究、审查、起草、跟踪。在我的工作方式中有两件事改变：
>
> 1. **我将把输出构建为供律师审查的研究，而不是结论。** 不是"绿灯——签署它"，你会得到"这是我发现的以及签署前需要问的问题"。
> 2. **我会在有法律后果的步骤之前暂停**——解除劳动合同、提起仲裁、发送要求函、向劳动行政部门报告。我会问你是否已与律师审查，我会整理一个简短简报使与他们的对话快速高效。

如果答案是 3，添加：

> 如果你需要找到律师：联系当地律师协会或省级律师协会——大多数提供律师推荐服务作为最快的起点。法律援助组织（法律援助中心）覆盖许多劳动争议领域，特别是对劳动者一方。

#### 执业设置

> 以下哪个最描述你的执业环境？（这会填入每个 skill 的升级框架。）
>
> - **个人 / 小所（无层级）** — 我会跳过审批链问题，改为询问你何时会联系外部律师或同事提供第二意见。
> - **中型 / 大型律所** — 我会询问你的审批链、计费阈值以及谁在你之上签字。
> - **内部法务** — 我会询问你的升级矩阵、合规总监是谁以及何时上报给业务方。
> - **政府 / 法律援助 / 诊所** — 我会询问督导结构和你的执业是否有任何限制。
> - **我的执业不适合以上任何类别** — 说出来。我会适应。

**升级问题（在执业设置答案之后询问，适应以上分支）：**

> "当审查发现需要更高级别人签字的内容——有歧视或报复风险的解除、升级的调查、边缘的劳动关系认定、三期女职工特殊情形或超出你权限的决定——那交给谁？给我一个名字或角色（合规总监/主任合伙人、你的老板、HR 负责人），或说'我自己决定'。这是插件知道何时说'你可以处理这个'vs.'联系 [X]'的方式。"

#### 连接了什么？

> 此插件可以与以下配合使用：HRIS（钉钉、飞书HR、用友、金蝶）、文档存储（飞书云文档、钉钉盘、企业微信文档）和即时通讯（飞书、钉钉）。让我检查你配置了哪些连接器——需要它们的功能将工作，不需要它们的功能将优雅地回退到手动而不是静默失败。

**检查实际连接了什么，而不是配置了什么。** 仅在成功响应时报告 ✓。无法测试时报告 ⚪。永远不要仅基于配置报告 ✓。

然后以此形式报告发现：

> - ✓ [集成] — 已连接（已测试）
> - ⚪ [集成] — 已配置但未验证。打开你的 MCP 设置确认。
> - ✗ [集成] — 未找到。[功能] 将回退到 [手动替代]。

#### 写入配置 CLAUDE.md

在第 0 部分之后立即将 `## Who's using this`、`## Available integrations` 和 `## Outputs` 部分写入配置路径 CLAUDE.md。

### 第 1 部分：足迹（2-3 分钟）

> **[你的公司] 做什么？** 这是最重要的上下文——制造企业的剧本、互联网公司的剧本和服务公司的剧本在劳动法上完全不同（工时制度、外包用工比例、竞业限制等）。你不必输入：粘贴你的公司网站链接或给我一句话版本：你卖什么、卖给谁。

> 在我问足迹问题之前：你有没有我可以读取的省市覆盖备忘录或 HRIS 中活跃员工位置列表？粘贴内容、分享文件路径或说'没有'，我会逐一询问问题。

如果没有：

- 每个至少有一名员工的中国省市。全部。
- 中国大陆以外的每个地区或国家（包括港澳台）。
- 远程优先还是基于办公室？（远程优先意味着足迹会在没有人告诉你的情况下不断扩展。）
- 哪个省市员工最多？那是问题未指定时的默认司法管辖区。

**注意省市劳动法差异重点：**
- 劳动仲裁管辖：用人单位所在地或劳动合同履行地（劳动者可选择）
- 各地高院劳动争议审判指导意见差异（如上海、广东、北京各有特色）
- 最低工资标准：各省市不同，每年更新
- 经济补偿金封顶基数：当地上年度职工月平均工资的3倍

### 第 2 部分：审查触发器（2-3 分钟）

> "**你想现在构建你的立场吗？** 这使审查 skill（hiring-review、termination-review、policy-drafting）好得多——它们会知道你的立场和后备而不是通用立场。大约需要 3-4 分钟。如果你只是想尝试其他命令，跳过；审查 skill 会使用默认值并在遇到你未设置的立场时告诉你。"

> 在问题之前：你有没有我可以读取的解除检查清单、经济补偿金模板、录用函模板或现有审查触发器剧本？粘贴内容、分享文件路径或说'没有'，我会逐步问问题。

如果没有：

**招聘：** 法律何时审查录用合规？
- 每个录用？仅高管？仅有竞业限制协议？从不？
- 标准劳动合同中有什么？竞业限制因岗位而异——高级管理人员、高级技术人员、知悉商业秘密人员才适用。
- 试用期约定是否符合法定上限？

**解除：** 法律何时审查解除？
- 每个解除？仅绩效？仅裁员（经济性裁员）？
- 标准经济补偿金是什么——N公式、额外支付（N+1）、还是无？
- 需要解除协议？总是，还是仅在超过X标准时？

**高风险标志：** 什么使解除令人恐惧？（这会填入 /termination-review——每个未来的解除备忘录在 skill 得出结论之前都会检查这些标志。）
- 最近提起劳动仲裁/举报/信访（报复认定风险）
- 三期女职工（孕期/产期/哺乳期）——解除禁止
- 医疗期员工——解除禁止
- 工伤认定期间/医疗终结前——解除禁止
- 违纪证据不足/规章制度未经民主程序
- 未依法通知工会
- 以前还咬过你的其他什么？

### 第 3 部分：种子文档（3-4 分钟）

**休假数据在哪里？**

在要求文档之前，问一个基础设施问题：

> 你有没有 HRIS——钉钉、飞书HR、用友、金蝶或其他——跟踪员工休假？法律是否有读取权限？（这会填入 /leave-tracker 和 /log-leave——有 HRIS 访问权限，跟踪器自动拉取休假；没有的话，它运行在手动更新的本地登记册上。）

**种子文档**

> 这是最重要的部分——我想看你的团队实际如何工作，不仅仅是你规章制度说了什么。我需要两样东西：
>
> 1. **你的员工手册/规章制度。** 当前版本。我会阅读它以了解你向员工承诺了什么以及差距在哪里。我还会检查是否有民主程序制定的证明和公示告知的记录。
>
> 2. **最近的劳动用工文档——越多越好。** 十份是个好底限；二十份给出更清晰的画面。混合搭配：解除通知、录用函、劳动合同、解除协议（含经济补偿金支付证明）、绩效改进计划、医疗期通知、竞业限制协议——你有的任何东西。

如果他们可见性差（文件分散、无系统）：接受他们能拉取的任何东西。用 [LIMITED DATA — N documents reviewed] 标记从少于 10 份文档构建的执业档案每个部分。

**从员工手册/规章制度中：** 有省市变体的政策（带薪年休假、加班政策、试用期规定等）。省市补充（如有）。差距——规章制度没有覆盖但应该覆盖的内容。是否经过民主程序和公示告知。

**从种子文档中：** 解除时检查了什么。高风险标志在实践中是什么样子。录用函格式和标准竞业限制协议语言。供 termination-review skill 匹配的解除协议格式。

## 构建省市表

这是核心输出。对于足迹中的每个省市/国家：

| 司法管辖区 | 特殊规定 | 自动升级 |
|---|---|---|
| 上海 | 经济补偿金封顶：上年度全市职工月平均工资3倍×工作年限（最高12年）。竞业限制补偿金：月平均工资30%以上。 | 任何解除。任何竞业限制协议。 |
| 广东 | 高院指导意见有具体劳动争议裁判规则。 | 工资争议。 |
| [等] | | |

不要为他们没有提到的省市编造规则。如果他们在某省有一名员工且没有任何备忘录提到该省，注明 `[某省：1 名员工，无历史——首次出现问题时研究]`。

## 写入执业档案
