# 产品法务 Plugin（中国法适配版）

中国互联网产品法务工作流：发布审查、营销声明审查、功能风险评估和快速"这是个问题吗？"分类。围绕从你实际发布审查历史中学到的风险校准构建——在*你的*公司阻断什么，而不是通用的。适配中国大陆法律体系，覆盖 PIPL/DSL/CSL 三法核心义务、算法推荐备案、生成式 AI 合规、等保 2.0、跨境数据传输、平台经济监管等中国特有产品法务场景。

**每个输出都是供律师审查的草稿——带引用、标记和把关——而不是法律结论。** plugin 完成工作：阅读文档、应用你的剧本、发现问题、起草备忘录。律师审查、验证并决定。引用按来源标记，以便你知道哪些来自元典，哪些需要检查。保密标记保守应用，因此不会意外放弃任何东西。重要行动——提交、发送、执行——在明确确认后进行。

## Who this is for

| 角色 | 主要工作流 |
|---|---|
| **产品法务/法务专员** | 发布审查、功能风险评估、校准维护 |
| **产品经理** | "这是个问题吗？"分类自助服务 |
| **市场营销** | 发布前的声明审查（广告法合规、绝对化用语审查） |
| **总法律顾问/法务总监** | 升级项目的功能风险评估 |

## First run: the cold-start interview

连接到你的发布跟踪器（飞书项目/Jira/Linear/禅道），阅读你过去的十个发布审查，了解你实际阻断什么与你放行什么。构建每个其他 skill 都读取的风险校准表。

你的配置存储在 `~/.claude/plugins/config/claude-for-legal/product-legal/CLAUDE.md`，并在 plugin 更新后保留。

```
/product-legal:cold-start-interview
```

## Commands

| 命令 | 功能 |
|---|---|
| `/product-legal:cold-start-interview` | 冷启动访谈 |
| `/product-legal:launch-review [PRD or ticket]` | 根据你的框架进行完整发布审查 |
| `/product-legal:marketing-claims-review [copy]` | 营销声明审查（含广告法绝对化用语检查） |
| `/product-legal:is-this-a-problem [question]` | 快速"这是个问题吗？"答案 |
| `/product-legal:matter-workspace` | 管理事项工作区（仅多客户私人执业）——新建、列表、切换、关闭、无 |

## Skills

| Skill | 目的 |
|---|---|
| **cold-start-interview** | 从访谈 + 过去发布审查编写 ~/.claude/plugins/config/claude-for-legal/product-legal/CLAUDE.md |
| **launch-review** | 按类别审查，针对你的公司校准；含中国特有场景检查清单 |
| **marketing-claims-review** | 声明分类：夸张/事实/比较/暗示/绝对；覆盖广告法、消费者权益保护法执法口径 |
| **feature-risk-assessment** | 当发布审查不够时，对一个问题进行深入探讨 |
| **is-this-a-problem** | 快速回答产品经理的飞书/钉钉问题 |
| **matter-workspace** | 为多客户执业创建、列出、切换和关闭事项工作区；隔离每个客户/事项，因此上下文不会在它们之间泄露 |

## Interactive commands vs. scheduled agents

上面的命令在你调用时运行——用于当你处理事项时。下面的 agent 按计划运行——用于当你不注意时发生变化的内容：

| Agent | 监视内容 | 默认节奏 |
|---|---|---|
| **launch-watcher** | 发布跟踪器中可能需要法律审查的即将发布；根据校准表过滤未来 30 天内有发布日期的工单 | 每日 |

## 中国法律依据

此 plugin 的审查框架以下列法律法规为核心依据：

**个人信息与数据合规**
- 《个人信息保护法》（PIPL，2021）——核心个人信息保护法，含敏感个人信息（第28条）、数据出境（第38-43条）、受托处理者协议（第21条）
- 《数据安全法》（DSL，2021）——数据分类分级、重要数据保护、数据出境申报（第36条）
- 《网络安全法》（CSL，2017）——网络安全等级保护（等保 2.0）、网络产品与服务安全、关键信息基础设施

**产品与消费者**
- 《产品质量法》——产品缺陷责任
- 《民法典》侵权责任编第1202-1207条——产品责任
- 《消费者权益保护法》第26条——格式条款（用户协议）公平性要求
- 《电子商务法》第17-18条——"大数据杀熟"、个性化推荐告知

**算法与 AI**
- 《互联网信息服务算法推荐管理规定》（2022）——算法推荐备案与用户权利
- 《互联网信息服务深度合成管理规定》（2023）——深度合成内容标识
- 《生成式人工智能服务管理暂行办法》（2023）——AIGC 服务备案与安全评估

**平台与内容**
- 《互联网平台分类分级指南》——超大/大型平台义务
- 《平台经济反垄断指南》——垄断行为（SAMR 执法）
- 《网络直播营销管理办法》——直播带货合规
- 《移动互联网应用程序信息服务管理规定》（2022）——App 合规

**知识产权**
- 《著作权法》（2020 修正）——版权、用户生成内容
- 《信息网络传播权保护条例》第14-17条——避风港/通知删除

**广告与营销**
- 《广告法》——禁止绝对化用语（第24条）、比较广告规则
- 《互联网广告管理办法》（2023）

**无障碍**
- 《无障碍环境建设法》（2023）——App 无障碍功能要求

**争议解决**
- 公司注册地/主要营业地人民法院诉讼
- 中国国际经济贸易仲裁委员会（CIETAC）
- 北京仲裁委员会（BAC）/ 上海国际经济贸易仲裁委员会（SHIAC）/ 华南国际经济贸易仲裁委员会（SCIA）

## Integrations

**首先连接研究工具——引用护栏依赖它。** 没有它，每个引用都被标记为 `[模型知识—待核验]`，并且每个可交付成果上方的审阅者注释记录来源未经验证。Skill 都可以工作；元典只是将验证工作从你的盘子上移开。

附带在 `.mcp.json` 中配置的连接器：

- **Slack** — 搜索消息、阅读频道、在你的工作区中查找讨论（通用存储桶）
- **Google Drive** — 从 Google Drive 搜索、阅读和获取文档（通用存储桶）
- **Linear** — 问题跟踪和项目管理
- **Atlassian** — Jira 问题和 Confluence 页面
- **Asana** — 任务和项目跟踪

元典（chineselaw MCP）在系统级配置中全局可用——法条检索、案例检索、企业信息查询。

连接跟踪器后：冷启动提取发布历史，发布审查提取工单上下文，发布观察 agent 监视日历。

## Quick start

```
/product-legal:cold-start-interview
```

然后：

```
/product-legal:is-this-a-problem "我们的算法推荐功能需要做什么合规备案？"
```

→ 根据你的风险表的即时答案。

```
/product-legal:launch-review PROJ-1234
```

→ 完整审查，按类别，带行动项。

## How it learns

你在 `~/.claude/plugins/config/claude-for-legal/product-legal/CLAUDE.md` 的执业档案不是静态的——它会随着你使用 plugin 而改进。Skill 会告诉你何时输出使用了你应该调整的默认值。你可以重新运行设置、直接编辑文件，或者告诉 skill 记录新立场。

## Notes

- 校准表就是全部。如果错了，每次审查都错。当你的风险姿态改变时（新监管机构政策、新执法案例、新 GC），重新运行设置。
- `is-this-a-problem` 是为产品经理自助服务设计的。它快速回答，并在应该时路由到真实审查。
- 功能风险评估是为 10% 需要深度的发布准备的。大多数不需要——不要生成文书工作。
- 中国互联网产品监管每年均有重大更新，plugin 的时效性依赖元典检索，不要依赖训练知识作为最终依据。

## Prerequisites

一些功能引用外部集成（文档管理、发布跟踪器、法律法规数据库）。这些不捆绑——如果你在环境中有这些中的一个的 MCP 服务器，相关功能将使用它。没有它，plugin 回退到文件上传和手动工作流。运行 `/product-legal:cold-start-interview --check-integrations` 以查看你的环境中可用什么。

## Configuration

你的配置存储在 `~/.claude/plugins/config/claude-for-legal/product-legal/CLAUDE.md`，并在 plugin 更新后保留——你只需运行一次设置。
