# /dsar-response（个人权利请求响应）

1. 加载执业档案 → 个人权利请求响应流程（系统清单、验证方法、响应时限）。
2. 运行以下工作流。
3. 分类请求类型。检查升级触发条件——如果触发任何条件，在继续之前路由。
4. 处理：验证身份 → 遍历系统清单 → 豁免分析 → 起草回复。
5. 输出回复草案。**不要发送**——由人工审查并发送。
6. 按内部流程记录个人权利请求。

**在粘贴请求之前：** 请求将包含个人信息。确认你的会话和输出存储满足你的数据处理要求。编辑你不需要的内容（证件附件、无关邮件记录）。不要在文件名中存储请求人姓名。

# 个人权利请求响应

## 事项上下文

**事项上下文。** 检查执业级 CLAUDE.md 中的 `## 事项工作区`。如果 `已启用` 为 `✗`，跳过本段的其余部分。

---

## 升级触发条件（在其他一切之前检查）

以下任一情形立即升级，在继续之前路由：

- 请求来自于已知或疑似涉及执法/监管调查的个人
- 请求牵涉未解决的监管机构投诉
- 请求涉及大量个人信息（可能表明批量拉取）
- 请求描述了数据泄露或安全事件（→ 同时触发安全事件响应流程）
- 个人明确提及准备向监管机构投诉或提起诉讼

如果以上任何触发，停止并向执业档案中的升级联系人报告。

## 第 1 步：分类请求类型

PIPL 规定的个人权利包括（第44-50条）：

| 权利类型 | PIPL 条款 | 典型请求描述 |
|---|---|---|
| 查阅权/复制权 | 第45条 | "我想查看你们持有的我的个人信息" / "请提供我的数据副本" |
| 更正权/补充权 | 第46条 | "我的信息不准确，请更正" / "请补充以下信息" |
| 删除权 | 第47条 | "请删除我的个人信息" / "我要求注销账号并删除所有数据" |
| 转移权 | 第45条第3款 | "请将我的数据转移给另一家服务商" |
| 撤回同意 | 第15条 | "我撤回之前对[X]的同意" |
| 注销账号 | 网络数据安全管理条例第23条 | "请注销我的账号" |
| 限制处理 | 第44条 | "在争议解决前请暂停处理我的数据" |
| 要求解释说明 | 第48条 | "请解释你们如何使用我的个人信息" |
| 拒绝接收推送/个性化广告 | PIPL 24条 | "请关闭针对我的个性化推荐" |

此请求类型：**[分类]**

## 第 2 步：身份验证

在处理任何请求之前，验证请求人身份。

**执业档案 → 个人权利请求响应流程 → 身份核验方式**：[PLACEHOLDER]

常用身份核验方式：
- 手机号码（短信验证码）
- 邮箱地址（邮件验证码）
- 账号登录验证
- 身份证号后四位核对（谨慎，避免额外收集个人信息）

**注意：** 核验过程本身不得额外收集超出必要范围的个人信息（PIPL 最小必要原则）。`[模型知识—待核验]`

**如果身份无法核验：** 起草"无法核验身份"拒绝信，说明需要请求人提供哪些可以核验的信息。

|---|---|---|---|

**对于删除请求：** 检查每个系统的技术可行性和是否有法定保留要求（如财务记录、合同记录等有法定保存期限的数据）。

**不确定的系统：** 标记 `[审阅]`——需要技术团队确认是否持有该个人的信息。

## 第 4 步：豁免分析

PIPL 规定的可以拒绝或延迟的情形（第18条）：

| 豁免情形 | 是否适用 | 说明 |
|---|---|---|
| 法律、行政法规规定应当保密 | [是/否] | [说明] |
| 法律、行政法规规定不需要告知 | [是/否] | [说明] |
| 可能妨碍国家机关履行法定职责 | [是/否] | [说明] |

**删除权的限制（PIPL 47条第2款）：**
- 法律、行政法规规定的保存期限未届满，不得删除 → 告知不能立即删除，停止除存储和安全保护外的处理
- 删除个人信息从技术上难以实现 → 停止除存储和安全保护外的处理，并告知

**转移权的条件（网络数据安全管理条例第25条）：**
- 能够验证请求人真实身份
- 请求转移的是本人同意提供的或基于合同收集的个人信息
- 转移具备技术可行性
- 转移不损害他人合法权益

如果转移请求存在次数等明显超出合理范围的情形，可以根据实际成本收取必要费用。

**拒绝须说明理由（PIPL 50条）：** 拒绝个人行使权利的请求，必须说明理由。

## 第 6 步：记录个人权利请求

按内部流程记录，内容包括：
- 请求收到日期
- 请求类型
- 请求人身份核验方式和结果
- 处理结果（满足/部分满足/拒绝）
- 回复发送日期
- 是否涉及监管机构投诉或诉讼

记录须妥善保存，以便在监管机构检查时证明已履行个人权利响应义务。

---

## 响应时限说明

PIPL 要求"及时提供"，无明确日历天数。`[模型知识—待核验实践中的合理时限]`

网络数据安全管理条例第23条规定"及时受理，提供便捷支持"。

实践中，通常参考15-30个工作日内完成响应作为合理时限，但须结合请求复杂程度调整。

涉及大量系统查询或复杂删除操作时，应先发送确认收到请求的通知，说明预计处理时间。

---

## 此 skill 不做什么

- 它不执行实际的数据删除或系统操作——它起草响应和操作指引，由技术/运营团队执行。
- 它不决定是否豁免——它提供豁免框架，由律师判断。
- 它不发送任何内容——所有回复须人工审查后发送。
