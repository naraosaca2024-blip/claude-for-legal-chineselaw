# /use-case-triage（处理活动合规分类）

1. 阅读执业档案。确认数据合规执业已配置——如未配置，停止并引导至设置。
2. 运行以下工作流。如不清楚，澄清活动内容。
3. 数据分类分级检查 → 个人信息检查 → PIPIA触发检查 → 个人信息处理规则冲突检查。
4. 输出：分类结论（可继续 / 需PIPIA / 需专项合规评估 / 停止）、推理、条件表（如需要）、跨 skill 移交。
5. 如需要PIPIA，提议继续进入 PIPIA 生成。

# 处理活动合规分类

## 事项上下文

**事项上下文。** 检查执业级 CLAUDE.md 中的 `## 事项工作区`。如果 `已启用` 为 `✗`，跳过本段的其余部分。

---

## 第 1 步：了解处理活动

如果描述不够清晰，询问（一次最多3个问题）：

1. **这个活动处理什么数据？** 具体是哪些类型的个人信息或其他数据？
2. **处理目的是什么？** 数据用来做什么？
3. **数据会和谁共享或在哪里处理？** 是否会发给第三方？是否会出境？

---

## 第 2 步：数据分类分级检查（DSL/网络数据安全管理条例）

依据《数据安全法》第21条和《网络数据安全管理条例》第5条，对数据进行分类分级：

| 数据类型 | 定义 | 此活动是否涉及 |
|---|---|---|
| **核心数据** | 关系国家安全、国民经济命脉、重要民生、重大公共利益等数据（DSL 21条） | [是/否/不确定] |
| **重要数据** | 特定领域/群体/区域或达到一定精度和规模，泄露/破坏后可能直接危害国家安全/经济运行/社会稳定/公共健康安全的数据（网络数据安全管理条例 62条） | [是/否/不确定] |
| **一般数据** | 上述两类之外的数据 | [是/否] |

**如果涉及核心数据：** 🔴 停止——须专项法律咨询，适用最严格的管理制度（DSL 21条）。`[审阅]`

**如果涉及重要数据：**
- 是否已识别/申报为重要数据？
- 提供/委托处理/共同处理重要数据前须进行风险评估（网络数据安全管理条例 31条）
- 须每年度开展风险评估并向主管部门报送（网络数据安全管理条例 33条）
- 出境须通过安全评估申报（网络数据安全管理条例 37条）

---

## 第 3 步：个人信息检查（PIPL）

**此活动是否处理个人信息？**（个人信息 = 以电子或其他方式记录的与已识别或可识别自然人有关的各种信息，PIPL 4条）

| 子检查 | 结果 |
|---|---|
| 是否处理个人信息？ | [是/否] |
| 如是：是否涉及**敏感个人信息**（生物识别/宗教信仰/特定身份/医疗健康/金融账户/行踪轨迹/未成年人）？ | [是/否/部分] |
| 是否处理未满14周岁未成年人个人信息？ | [是/否] |
| 处理规模是否达到规定数量（目前参考阈值：100万人以上涉及出境安全评估，1000万人以上须适用重要数据额外规定）？ | [是/否/不确定] `[待核验最新门槛]` |

**如果处理敏感个人信息：**
- 须有特定目的和充分必要性（PIPL 28条）`[已确认 PIPL 28条]`
- 须取得单独同意（PIPL 29条）`[已确认 PIPL 29条]`
- 须告知处理敏感个人信息的必要性及影响（PIPL 30条）`[已确认 PIPL 30条]`

**如果处理未成年人个人信息（14周岁以下）：**
- 须取得父母/监护人同意（PIPL 31条）`[已确认 PIPL 31条]`
- 须制定专门处理规则（PIPL 31条）`[已确认 PIPL 31条]`

## 第 5 步：数据出境检查（如涉及出境）

如果处理活动涉及向境外提供数据：

| 情形 | 须采用的机制 |
|---|---|
| CIIO 向境外提供任何数据 | 安全评估申报 |
| 处理≥100万人PI向境外提供 | 安全评估申报 |
| 累计向境外提供≥10万人PI或≥1万人敏感PI | 安全评估申报 |
| 向境外提供重要数据 | 安全评估申报 |
| 不触发上述门槛的个人信息出境 | 标准合同备案 |
| 合同履行/劳动管理/法定义务/紧急情况 | 例外情形（无需评估/合同/认证） |

## 第 6 步：个人信息处理规则一致性检查

此处理活动是否与已发布的个人信息处理规则（隐私政策）一致？

| 检查项 | 状态 |
|---|---|
| 处理规则中是否已列明此处理目的？ | [✅/⚠️未列明/❌未告知] |
| 处理规则中是否已列明此类个人信息？ | [✅/⚠️/❌] |
| 处理规则中是否已列明相关第三方/受托方？ | [✅/⚠️/❌] |

## 跨 skill 移交

- **需要 PIPIA** → 提议运行 `/privacy-legal:pia-generation`，携带本次分类结论作为上游背景
- **需要合规差距分析** → 提议运行 `/privacy-legal:reg-gap-analysis`（检查具体法规要求）
- **已发现处理规则冲突** → 提议运行 `/privacy-legal:policy-monitor`（更新处理规则）
- **涉及数据出境** → 提议专项出境合规评估（元典检索最新出境规则）

---

## 此 skill 不做什么

- 它不确定法律适用——它识别应检查的法律框架，由律师判断。
- 它不设计产品——它识别合规要求，由产品团队决定如何满足。
- 它不做最终合规结论——所有 `[审阅]` 标记须律师确认。
